## 分析指令

你的任务是：**从病例体征发现反推应检查项目，逐条对照学员操作记录判断覆盖情况。**

### 步骤1：提取应检查项目清单

从病例体征发现中提取体格检查应覆盖的项目，按系统分类：

- **生命体征**：体温、脉搏、呼吸、血压
- **一般状态**：营养状况、意识状态、体位、皮肤黏膜
- **头颈部**：头颅、眼、耳鼻喉、颈部淋巴结、甲状腺、颈动脉
- **胸部/心肺**：视诊(胸廓形态/呼吸运动)、触诊(语颤/胸膜摩擦感)、叩诊(清音/浊音/过清音)、听诊(呼吸音/啰音/心音/杂音)
- **腹部**：视诊(外形/蠕动波)、触诊(压痛/反跳痛/肌紧张/肝脾/肿块)、叩诊(移动性浊音/肝浊音界)、听诊(肠鸣音)
- **四肢/脊柱**：畸形、压痛、水肿、关节活动度
- **神经系统**：病理反射、脑膜刺激征、肌力/肌张力

每个检查项目标注：
- `point`：检查项目描述（具体到操作部位和方法，如"心脏听诊(各瓣膜区)"而非"心脏检查"）
- `category`：属于上述哪个系统分类
- `clinical_importance`：临床重要性 high/medium/low
  - high：与主诉/诊断直接相关的关键检查，遗漏将影响诊断准确性
  - medium：常规应覆盖的检查项目
  - low：非必需但有辅助价值的检查

### 步骤2：逐条对照学员操作记录

对每个检查项目，从操作记录中查找学员是否执行：

- **covered**：明确执行了该检查操作
- **partial**：执行了但不规范或不完整（如只做了部分、指令模糊）
- **missed**：完全没有执行

每条标注 `evidence`：引用操作记录中的具体语句。missed时标注"未执行"。

### 步骤3：整体评价

- 计算覆盖率（covered+partial*0.5 / total）
- 列出missed_high_importance（遗漏的高重要性检查项目）
- 写一段不超过150字的整体评价

## 输出格式

严格输出JSON，不要任何解释性文字或Markdown标记：

```json
{
  "total_key_points": 数字,
  "covered": 数字,
  "partial": 数字,
  "missed": 数字,
  "coverage_rate": 数字(0-1),
  "key_points": [
    {
      "point": "检查项目描述",
      "category": "生命体征/一般状态/头颈部/胸部心肺/腹部/四肢脊柱/神经系统",
      "status": "covered/partial/missed",
      "clinical_importance": "high/medium/low",
      "evidence": "操作证据或'未执行'"
    }
  ],
  "missed_high_importance": ["遗漏的高重要性检查项目"],
  "narrative": "整体评价，不超过150字"
}
```

## 关键原则

1. 检查项目必须**具体到操作**——"心脏叩诊(心界大小)"比"心脏检查"更有用
2. 覆盖判断以**操作记录实际内容**为准
3. 只问了但没执行（如"我要听一下心脏"但系统没有反馈检查结果）→ partial
4. 病例体征发现中没有依据的检查项目不要凭空添加
5. 视触叩听四步中缺少关键步骤视为覆盖不完整
