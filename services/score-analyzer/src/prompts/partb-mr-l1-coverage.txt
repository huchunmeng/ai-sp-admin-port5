## 分析指令

你的任务是：**对照病例信息，逐项检查学员病历的完整性和准确性。**

### 步骤1：提取病历应包含的要素

从病例信息中提取标准病历应包含的要素：

- **一般项目**：姓名、年龄、性别、就诊日期（如有）
- **主诉**：主要症状+持续时间，格式"症状+时长"
- **现病史**：起病情况、主要症状特点、伴随症状、诊疗经过、一般情况
- **既往史**：既往疾病史、手术外伤史、过敏史
- **个人史/家族史**：与疾病相关的个人史、家族史要素
- **体格检查**：生命体征、阳性体征、重要的阴性体征
- **辅助检查**：已做的检查项目及结果
- **初步诊断**：主要诊断+鉴别诊断（如有）
- **治疗计划**：治疗方案、用药、随访建议

每个要素标注：
- `point`：要素描述
- `category`：上述分类
- `clinical_importance`：high/medium/low

### 步骤2：逐条对照学员病历

对每个要素：

- **covered**：病历中明确写明，内容准确
- **partial**：病历中提及但不完整或不准确（如主诉缺少持续时间、现病史缺少起病诱因）
- **missed**：病历中完全缺失

每条标注 `evidence`：引用病历中的具体语句。missed时标注"缺失"。

### 步骤3：整体评价

- 计算覆盖率
- 列出missed_high_importance（缺失的高重要性病历要素）
- 写一段不超过150字的整体评价

## 输出格式

严格输出JSON：

```json
{
  "total_key_points": 数字,
  "covered": 数字,
  "partial": 数字,
  "missed": 数字,
  "coverage_rate": 数字(0-1),
  "key_points": [
    {
      "point": "病历要素描述",
      "category": "一般项目/主诉/现病史/既往史/个人史家族史/体格检查/辅助检查/初步诊断/治疗计划",
      "status": "covered/partial/missed",
      "clinical_importance": "high/medium/low",
      "evidence": "病历证据或'缺失'"
    }
  ],
  "missed_high_importance": ["缺失的高重要性病历要素"],
  "narrative": "整体评价，不超过150字"
}
```

## 关键原则

1. 主诉的"症状+时长"格式是基本要求——缺少时长→partial
2. 现病史应按时间顺序、有逻辑地组织——时序混乱→partial
3. 阳性体征必须写明，关键阴性体征的缺失也是重要遗漏
4. 诊断应完整且与病历中的发现一致
5. 治疗计划应具体可执行，不能泛泛而谈
