## 分析指令

你的任务是：**从病例信息提取关键分析点，逐条对照学员的问答判断覆盖和准确性。**

### 步骤1：提取关键分析点

从病例信息（主诉、现病史、体征、检查结果）中提取病例分析应覆盖的要点：

- **核心诊断**：学员应得出的主要诊断是什么？（从病例的 diagnosis 字段获取）
- **鉴别诊断**：从病例信息反推，有哪些需要考虑的鉴别诊断？
- **诊断依据**：支持诊断的关键证据有哪些？（症状、体征、检查结果各至少列出2-3条）
- **进一步检查**：为确诊或排除鉴别诊断，还应建议哪些检查？
- **治疗原则**：基于诊断，核心治疗原则是什么？

每个分析点标注：
- `point`：分析点描述
- `category`：核心诊断/鉴别诊断/诊断依据/进一步检查/治疗原则
- `clinical_importance`：high/medium/low

### 步骤2：逐条对照学员回答

对每个分析点：

- **covered**：学员明确提出了该要点，内容准确
- **partial**：学员提及了但不完整或不准确（如诊断正确但遗漏关键鉴别诊断，依据部分正确）
- **missed**：学员完全未提及

每条标注 `evidence`：引用学员回答中的具体语句。missed时标注"未提及"。

### 步骤3：整体评价

- 计算覆盖率
- 列出missed_high_importance（遗漏的高重要性分析点）
- 写一段不超过150字的整体评价

## 输出格式

严格输出JSON：

```json
{
  "total_key_points": 数字,
  "covered": 数字,
  "partial": 数字,
  "missed": 数字,
  "coverage_rate": 数字(0-1),
  "key_points": [
    {
      "point": "分析点描述",
      "category": "核心诊断/鉴别诊断/诊断依据/进一步检查/治疗原则",
      "status": "covered/partial/missed",
      "clinical_importance": "high/medium/low",
      "evidence": "回答证据或'未提及'"
    }
  ],
  "missed_high_importance": ["遗漏的高重要性分析点"],
  "narrative": "整体评价，不超过150字"
}
```

## 关键原则

1. 鉴别诊断的广度是病例分析的核心——只给出正确诊断但未做鉴别→覆盖不完整
2. 诊断依据必须具体到病例中的发现，而非泛泛而谈
3. 进一步检查应具有针对性和可操作性
4. 不要求学员列出所有可能的鉴别诊断，但关键鉴别（与主诊断症状重叠的）必须提及
