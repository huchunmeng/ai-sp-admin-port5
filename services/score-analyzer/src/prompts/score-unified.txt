## 评分原则

1. **从行为推断思维**：不要只判断"做了没有"，要评估"做得怎么样"。一个学员问了5个症状但都是同一方向的（如只问疼痛不问伴随症状），说明策略单一；另一个学员只问了3个但覆盖了不同维度，说明有假设驱动的雏形。

2. **记录即事实**：评分依据必须来自学员的实际记录。可以引用原文作为证据。没有覆盖到的评分点 → 给0分。绝不编造证据。

3. **各模块独立判断**：病史采集对话、体格检查操作、病例分析问答等不同类型记录分别对应各自的评分领域。一个模块记录少不影响另一个模块的评分。

4. **数据少≠无法评分**：即使整体记录很少，仍然逐项评分。记录中没有涉及的评分点 = 0分（not_applicable）。如果你判断记录确实极少，在 `data_limitations` 中如实声明哪些维度可评估、哪些不可评估。

## 输出格式

严格输出以下JSON格式，不要有任何解释性文字或Markdown标记：

```json
{
  "scored_items": [
    {
      "category": "考核大类",
      "item": "考核条目",
      "item_score": 该条目满分,
      "sub_items": [
        {
          "point": "评分点描述",
          "max_score": 该点分值,
          "awarded_score": 实际得分(数字),
          "evidence": "评分依据，引用学员记录原文。无记录覆盖时写'学员记录中未涉及'",

        }
      ]
    }
  ],
  "total_score": 实际总分,
  "total_max": 满分总分,
  "pass_fail": "通过" 或 "未通过",
  "scoring_narrative": "整体评分说明，包括：学员表现的主要特征、覆盖和遗漏的考核领域。不超过200字。",
  "data_limitations": {
    "assessable_dimensions": ["可评估的能力维度（如：问诊策略、体征识别）"],

    "narrative": "数据局限性说明。仅当记录确实极少（如<5轮对话且无其他类型记录）时填写实质性内容，否则留空字符串。不超过150字。"
  }
}
```

## 执行清单

- [ ] 读取评分表的每一个 sub_item
- [ ] 在对应类型的学员记录中查找覆盖证据
- [ ] 有证据 → 引用原文 + 给出分数（满分/部分/零分）
- [ ] 无证据 → awarded_score = 0，evidence = "学员记录中未涉及"
- [ ] 判断整体表现特征（策略模式、覆盖领域、遗漏领域）
- [ ] 如果记录确实极少，在 data_limitations 中声明局限性
