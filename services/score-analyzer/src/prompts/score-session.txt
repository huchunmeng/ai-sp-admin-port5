## 评分指令

对上述评分表的每一个 sub_item 进行独立评分：

1. **对照对话记录**：逐条检查 sub_item 的 point/rules，判断学员是否在对话中覆盖了该评分点
2. **评分标准**：
   - 满分：学员主动、明确地覆盖了该评分点，提问清晰、追问到位
   - 部分分(50-70%)：学员涉及了该评分点但不够深入或不够完整
   - 零分：学员完全没有涉及该评分点
3. **证据提取**：为每个评分点提供对话中的具体证据（引用学员提问或SP回答的原文片段）
4. **关联对话**：标注与评分相关的对话轮次编号（sequence字段值）

## 输出格式

严格输出以下JSON格式，不要有任何解释性文字或Markdown标记：

```json
{
  "scored_items": [
    {
      "category": "考核大类",
      "item": "考核条目",
      "item_score": 该条目满分,
      "sub_items": [
        {
          "point": "评分点描述",
          "max_score": 该点分值,
          "awarded_score": 实际得分(数字),
          "evidence": "评分依据，引用对话原文",
          "related_turns": [关联的对话轮次sequence编号]
        }
      ]
    }
  ],
  "total_score": 实际总分(所有awarded_score之和),
  "total_max": 满分总分,
  "pass_fail": "通过" 或 "未通过",
  "scoring_narrative": "评分整体说明，简要指出主要扣分点和亮点，不超过150字"
}
```

## 评分原则

1. 以对话记录为准，不能假设学员"可能问了但没有记录"
2. 对明确问到的内容给予充分认可
3. 对部分涉及但不完整的给予部分分
4. 评分要严格但公正，不给"人情分"
5. 总分不超过满分
6. evidence字段必须引用对话原文，不可编造
7. related_turns使用对话记录中的sequence编号
