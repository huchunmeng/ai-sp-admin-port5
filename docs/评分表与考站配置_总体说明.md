### 目录结构

```
│   ├── 儿科-病历书写站.xlsx
│   ├── 儿科-临床思维站.xlsx
│   └── 儿科-交流沟通站.xlsx
├── ...           (共34个专业文件夹)
```

### 文件格式

每个Excel文件包含两个Sheet：

**Sheet 1 — 评分表**：三列表格

| 考核项目 | 评分项 | 分值 |
|----------|--------|------|
| 病史采集 — 现病史采集 | 发病时间与诱因 | 8 |
| 病史采集 — 现病史采集 | 主要症状特点 | 8 |
| ... | ... | ... |
| 合计 | | 100 |

- 第一列 = 考核项目名称 + 评分分类名称（如"病史采集 — 现病史采集"）
- 第二列 = 评分细项
- 第三列 = 分值（纯数字，已按专业考站项目分值比例缩放）

**Sheet 2 — 评分说明**：
- 考站概述（时长、总分、考核内容）
- 各考核项目的分类说明和评分要点
- 评分等级标准（优秀/良好/合格/不合格）
- 总评等级参考

### 分值缩放规则

模板中评分细项的标准分值（如"发病时间与诱因=8分"）按考站中对应项目实际分配分值等比缩放：
- 例如：内科接诊病人站病史采集分配30分，模板满分100，缩放因子=0.3，则8分项→2.5分
- 缩放后以0.5分为最小单位，差值从高分项向低分项调整

## 六、系统导入方案

### 数据文件

`docs/考站与评分表导入数据.json` — 包含三层实体及完整外键关联。

### 导入顺序

```
第1步: scoreTemplates (7条)
  id = templateCode (TPL-STD ~ TPL-STD-7)
  字段: name, description, totalScore, categories[].items[].grading

第2步: stations (24条)
  id = ST-001 ~ ST-024
  字段: name, duration, totalScore, projects[], scoreTableRefs[]
  FK:  scoreTableRefs[].templateCode → scoreTemplates.id

第3步: specialtyStationLinks (34条)
  key = 专业名称
  字段: stations[].stationId, stations[].projectTemplateMap[]
  FK:  stations[].stationId → stations.id
```

### FK关联链

```
scoreTemplates (id=templateCode)
     ↑ templateCode (FK)
stations.scoreTableRefs[]
     ↑ stationId (FK)
specialtyStationLinks[specialty].stations[]
```

### 文件索引交叉验证

`fileIndex` 数组记录了每个(专业, 考站)组合对应的Excel文件名：

```json
{
  "specialty": "内科",
  "files": [
    { "fileName": "内科-接诊病人站.xlsx", "stationName": "接诊病人站", "templateCodes": ["TPL-STD","TPL-STD-2","TPL-STD-6"] },
    ...
  ]
}
```

导入后可据此校验：Excel文件是否齐全、文件内模板数量是否与考站配置一致、模板评分项是否完整。

## 七、与国标的对标情况

| 维度 | 对标结果 |
|------|---------|
| 评分表模板 | ✅ 7个模板覆盖国标全部评分维度，大类/细项/分值均对齐 |
| 考站设置 | ✅ 34专业考站全部对齐，已补全5处缺失 |
| 评分等级 | ✅ 四档等级（优秀/良好/合格/不合格）与国标一致 |
| 考核项目 | ✅ 6大考核项目全覆盖，分值分配符合各专业特色 |
| 专业名称映射 | ✅ 13个专业对体格检查评分表有自定义名称（神经内科→神经系统检查，眼科→眼部检查等） |

### 已知差异（设计决策）

1. **总分归一化**：国标各专业各考站总分不完全一致（部分专业初诊站60分、沟通站40分等），当前系统统一为每站100分，通过项目分值权重体现差异。如需按国标原始分值导入，需调整 `residency.js` 中的 `score` 字段。

2. **评分等级分值映射**：当前使用等比缩放的等级分值（如优秀=100%满分、良好=80%、合格=60%、不合格=30%），与国标中部分固定分值存在微调空间。

## 八、维护与扩展指南

### 新增专业

1. 在 `residency.js` 中新增 `major` 条目，定义其考站和项目
2. 运行 `node scripts/gen-import-data.mjs` 重新生成导入JSON
3. 运行 `node scripts/gen-scoring-tables.mjs` 重新生成Excel文件
4. 更新本文档的统计数字

### 修改评分模板

1. 在 `score-tables/index.js` 中修改对应模板的 `categories`/`items`
2. 重新运行两个脚本
3. 注：模板修改会影响所有使用该模板的专业

### 修改考站项目分值

1. 在 `residency.js` 中修改对应 `station.projects[].score`
2. 重新运行两个脚本（Excel分值将自动重新缩放）
