## 2026-07-14

### 清华长庚多中心实证研究 —— 系统支持方案设计

**背景**：王老师提出4条需求（7/11），结合7/13周例会讨论结论，完成了实证研究系统支持方案的完整设计。

**王老师原始需求**：
1. 下周开始对未试用中心同步安装和试用
2. 建立专门病例库：按专业分类（医学生/内/外/妇/儿），每专业分训练库+考核库
3. 机构后续购买/继续试用可保留病例库
4. 学员每周训练3个病例，确定推送模式

**讨论过程与关键决策**：

| 轮次 | 关键决策 |
|------|---------|
| v1→v2 | 不建独立病种库，在现有库上扩展；不做研究定制功能 |
| v2→v3 | 训练/考核病例库已有（删除usage标签设计）；确定三阶段病种库方向 |
| v3.1→v3.2 | 三阶段病种库独立建设（院校教育/住培/专培各按各自规范）；专家审核病例库作为premium产品 |
| 7/13周例会 | 分机构隔离可行（当前系统做不到同平台细粒度权限隔离），无需开发复杂隔离功能 |
| 最终版 | 机构命名加【实证研究】前缀；病例授权功能替代组别标记；审核流程已有无需开发 |

**最终需求与方案**：

| # | 需求 | 类型 |
|:--:|------|:--:|
| 1 | 开通【实证研究】前缀试用机构，与已有机构隔离 | 运营 |
| 2 | 现有病种库基础上增加研究所需病种（优先复用） | 数据维护 |
| 3 | AI生成→编辑审核→专家审核→入库 | 运营（审核流程已有） |
| 4 | 病例授权功能：运营端为病例勾选授权机构，实现双向隔离 | 开发 |
| 5 | 教学任务功能：教师创建任务→指定学员→定时推送→可设及格线 | 开发 |

**产品开发量**：2项功能（病例授权 + 训练任务分配），预计7/21-8/1完成。
