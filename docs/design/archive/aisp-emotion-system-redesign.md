# AI-SP 情绪系统重构设计

> 文档版本：v7.0 | 日期：2026-06-10 | 状态：设计完成，待实施

---

## 一、背景与问题

当前情绪系统（v3.0，9维向量）存在以下结构性问题：

| # | 问题 | 现象 |
|---|------|------|
| 1 | **状态读取滞后** | LLM 调用前不更新状态，本轮攻击要到下一轮才有情绪反应 |
| 2 | **负面情绪触发阈值过高** | angry 基线0，单次攻击+2，衰减-0.3，阈值5→需连续3轮攻击才触发 |
| 3 | **所有攻击一视同仁** | "你是AI吗"和"滚"给相同的 angry:+2 |
| 4 | **情绪消退不真实** | 中性提问就能自动消怒，不需要学生道歉 |
| 5 | **缺乏性格差异** | 所有SP对同一攻击做同样反应 |
| 6 | **只有愤怒有完整设计** | 恐惧、悲伤、焦虑仅定义了参数，无触发条件/行为模式/消退机制 |
| 7 | **LLM输出增量，方向正确但算术不准** | 3B模型不擅长精确计算+2还是+3 |

## 二、核心架构：两层分离

```
┌─────────────────────────────────────────────────────────────┐
│  职责边界                                                    │
│                                                              │
│  LLM 全权负责: 综合判断                                        │
│    - 看当前情绪向量 + 性格 + 学生这话                           │
│    - 判断学生意图（intent）                                    │
│    - 输出情绪绝对值 emotion_score（0-10）                      │
│    - 输出回复文本（语气与情绪同步）                              │
│    - 决定方向：该不该生气、该不该消气、到什么程度                │
│                                                              │
│  JS 状态机负责: 惯性约束                                       │
│    - 负面情绪上升 → 不限制（一句话可以让人炸毛/吓瘫/崩溃）       │
│    - 负面情绪消退 → 限制速度（不会秒好）                        │
│    - 峰值持续期 → 到达最高点后强制持续N轮，除非LLM判断该释放     │
│    - 自然回落 → 峰值结束后逐轮下降                              │
│    - 平台止步 → 降到某个临界值后停止，必须学生行动               │
│    - trust/rapport → 纯JS计算，不交给LLM                      │
└─────────────────────────────────────────────────────────────┘
```

### 关键原则

1. **LLM是意图和情绪的最终裁判**。它综合所有信息判断学生态度+SP应该处于什么情绪状态。
2. **状态机是刹车，不是方向盘**。它不决定情绪方向，只限制变化速度。
3. **预更新是纯时间驱动的**。它处理"时间流逝自然发生的事"（峰值计数/自然回落/被动消退），不依赖对学生的意图判断。
4. **每种情绪有独立的行为模式**。愤怒≠恐惧≠悲伤，触发条件不同、消退方式不同、需要的学生行动也不同。

## 三、情绪维度全景

### 3.1 九维分类

| 类别 | 维度 | 性质 | 触发来源 |
|------|------|------|---------|
| **对抗型** | angry | 学生态度驱动 | 被攻击/被否定/被催促/被敷衍 |
| **脆弱型** | fearful | 病情+态度驱动 | 重病提示/死亡联想/学生冷漠 |
| **脆弱型** | sad | 病情+关怀缺失 | 丧失感/不良预后/无人理解 |
| **波动型** | anxious | 基线状态 | 就诊本身的紧张，随时间自然消退 |
| **躯体型** | in_pain | 病例驱动 | 病例设定的躯体症状 |
| **正向型** | calm | 综合结果 | 有序对话、被尊重、被理解 |
| **正向型** | relieved | 事件驱动 | 收到好消息、被安抚成功 |
| **关系型** | trust | 纯JS计算 | 专业能力被认可的程度 |
| **关系型** | rapport | 纯JS计算 | 情感连接的质量 |

### 3.2 情绪交互矩阵

```
                  被攻击   被否定    被催促    被敷衍    被安抚    被告知坏消息   被重复问
angry             ↑↑↑      ↑↑       ↑        ↑        ↓↓         —            ↑
fearful           —        ↑        —        ↑        ↓          ↑↑           —
sad               —        ↑↑       —        ↑↑       ↓          ↑            ↑
anxious           ↑        ↑        ↑↑       —        ↓↓         ↑↑↑          ↑
in_pain           —        —        —        —        ↓          —            —
calm              ↓↓↓      ↓↓       ↓        ↓        ↑↑         ↓            ↓
relieved          —        ↓        —        —        ↑↑↑        ↓↓           —
trust             ↓↓       ↓        ↓        ↓        ↑          —            ↓
rapport           ↓↓↓      ↓↓       ↓        ↓↓       ↑↑         —            ↓↓
```

图例：↑↑↑ 强烈上升 | ↑↑ 明显上升 | ↑ 轻度上升 | — 基本不变 | ↓ 轻度下降 | ↓↓ 明显下降 | ↓↓↓ 强烈下降

### 3.3 各情绪的"学生需要做什么才能消退"

| 情绪 | 自然消退？ | 需要学生做什么 |
|------|-----------|--------------|
| **angry** | 不自然消 | 道歉 + 改变态度（intent=reassuring 持续多轮效果更好） |
| **fearful** | 不自然消 | 安抚+解释病情不严重+专业能力展示（intent=reassuring + 给出明确信息） |
| **sad** | 不自然消 | 共情+倾听+不催促（intent=reassuring + 开放式提问让患者倾诉） |
| **anxious** | 自然-0.3/轮 | 有序专业问诊本身就减轻焦虑；安抚加速消退 |
| **in_pain** | 不自然消 | 与病情相关，不受学生态度影响，问诊中自然提及疼痛相关问题时可能升高 |
| **calm** | 自然+0.1/轮 | 持续有序对话即可 |
| **relieved** | 自然-0.2/轮 | 新的好消息可重新提升 |

## 四、完整流程

```
第N轮学生消息
  │
  ├─ ① 状态机预更新（纯时间驱动，不依赖学生意图）
  │     • 峰值持续计数：处于峰值的维度，计数器+1
  │       若计数达到peakDuration → 释放峰值，执行首轮自然回落
  │     • 自然回落：峰值已释放的维度，按回落速率逐轮下降
  │       降到平台值后停止（等学生行动才能突破）
  │     • 被动自然消：低区间维度缓慢消退
  │       anxious -0.3/轮, relieved -0.2/轮, calm +0.1/轮
  │
  ├─ ② buildSystemPrompt（注入预更新后的情绪状态）
  │     • 当前情绪向量（9维精确值）
  │     • 情绪3轮趋势（从debugLog提取，↑↓→标注每个维度的变化方向）
  │     • 历史攻击记忆标记（曾经被惹毛过？→ 情绪触发×1.3）
  │     • 多轮无进展检测（学生兜圈子？→ 不耐烦加速累积）
  │     • 各维度梯度标签 + 文本指导
  │     • 性格描述 + 表达调节
  │     • 历史统计（连续攻击/安抚轮数）
  │     • 防重复列表（最近N条SP回复）
  │     • 情绪跨话题保持警告
  │
  ├─ ③ LLM 综合判断
  │    输入：系统提示词（含情绪上下文）+ 对话历史
  │    输出：{"text":"回复","emotion_score":{"angry":6,"fearful":4},"intent":"aggressive"}
  │    • emotion_score 是当前情绪绝对值(0-10)，不是增量
  │    • intent 由LLM全权判断
  │    • LLM可同时设多个维度（如愤怒+恐惧并存）
  │    • 变化不大的维度可不写
  │
  ├─ ④ 解析LLM输出 + 校验
  │    • repairJSON → parse
  │    • 提取 text, emotion_score, intent, deep_reassure
  │    • 校验（未通过校验的维度视为LLM未设）：
  │        - emotion_score每个维度值必须是0-10之间的数字
  │        - intent必须在 [reassuring,neutral,cold,pressuring,dismissive,aggressive] 内
  │        - deep_reassure仅intent=reassuring时可设，否则忽略
  │
  ├─ ⑤ 状态机采纳（惯性约束）
  │    对emotion_score中LLM明确设了值的维度：
  │      • 如果 target ≥ current → 直接采纳（上升不限制）
  │      • 如果 target < current → 惯性约束：
  │          - 先检查峰值：若当前在峰值且LLM intent=reassuring → 释放峰值
  │          - 应用 MAX_DOWN_PER_TURN 限制下降幅度
  │    对LLM未设值的维度：
  │      • 保持预更新后的值不变（已在步骤①处理过消退）
  │    然后更新：
  │      • trust/rapport（纯JS，基于LLM的intent）
  │      • 连续攻击/安抚计数器（基于LLM的intent）
  │      • 峰值计数器（基于最终向量值）
  │
  └─ ⑥ 存储消息 + 记录调试日志
```

### 流程对比

| 步骤 | 旧实现（v3.0） | 新设计（v4.0） |
|------|---------------|---------------|
| 预更新 | 仅被动消退（anxious-0.3等） | 峰值持续+自然回落+平台+被动消退 |
| 意图判断 | correctIntent() 正则覆盖LLM | LLM全权判断 |
| LLM输出 | 增量/绝对值混合 | 纯绝对值，可同时设多维度 |
| 状态机采纳 | 峰值+消退在LLM之后 | 预更新在LLM之前，采纳只做惯性约束 |
| 多情绪并存 | 未处理 | LLM可同时输出angry+fearful等组合 |
| 降级路径 | 正则意图+关键词匹配 | 已移除 |

## 五、各情绪详细设计

### 5.1 愤怒 angry

**触发条件（LLM自主判断）：**
- 被直接攻击："滚""闭嘴""你傻吗"
- 被否定/质疑："你是AI吧""假的吧""装的"
- 被催促："快点""别磨蹭"
- 被敷衍："嗯""哦"
- 被反复问同一问题（学生没在听）
- 信任/关系降到冰点后的反击

**行为特征：**
- 攻击性向外，语言变得尖锐、简短
- 配合度下降但仍在交流（与恐惧的"关闭"不同）
- 高rapport时愤怒表达更克制，低rapport时直接爆发

**梯度：**

| 区间 | 状态标签 | 文本指导 |
|------|---------|---------|
| 0–1.5 | — | 配合，无任何不悦，正常回答 |
| 1.5–3.5 | slightly_irritated | 回答变短一两字，语气里藏了一丝不痛快，但不点破 |
| 3.5–6.0 | annoyed | 明显敷衍，"你到底想问什么"，可能反问，但还在答 |
| 6.0–8.0 | angry | 直接表达不满，"你这样问我没法回答"，配合度很低 |
| 8.0–10 | furious | 高声质问或直接拒绝，"不看了！"，可能终止对话 |

**参数：**
- 上升：不限制
- 消退惯性：↓1.5/轮（rapport≥5时放宽到↓3.0/轮）
- 峰值阈值：≥8.0
- 平台止于：6.0
- 被动自然消：0（不消，必须学生道歉）

---

### 5.2 恐惧 fearful

**触发条件（LLM自主判断）：**
- 讨论重病/癌症/手术/死亡等话题
- 学生表情严肃/冷漠，缺乏共情
- 被告知检查结果可能不好
- 学生用专业术语碾压（听不懂但知道严重）
- 自身病情严重（病例基线高fearful）

**行为特征：**
- 防御性向内，退缩/语无伦次
- 可能反复问"严不严重""会不会死"
- 不像愤怒那样攻击学生，而是自我封闭
- 极度恐惧时崩溃，哭出来，需要安抚才能稳定

**梯度：**

| 区间 | 状态标签 | 文本指导 |
|------|---------|---------|
| 0–2 | — | 不提也不表现 |
| 2–4 | uneasy | 偶尔犹豫一下，但不会主动说害怕 |
| 4–6.5 | fearful | 断断续续，可能主动问"严不严重" |
| 6.5–9 | very_fearful | 声音发抖，语无伦次，需要安抚才能继续 |
| 9–10 | shut_down | 恐惧崩溃，边哭边说或泣不成声，需要安抚才能破冰 |

**参数：**
- 上升：不限制
- 消退惯性：↓1.0/轮
- 峰值阈值：≥9.0（关闭状态）
- 平台止于：4.0
- 被动自然消：0（不消，恐惧不会自己消失）
- **关键差异**：释放条件不是"道歉"而是"安抚+给出专业解释"（LLM判断intent=reassuring + 学生给出明确信息）

---

### 5.3 悲伤 sad

**触发条件（LLM自主判断）：**
- 谈及丧失（亲人去世/失去健康/失去工作能力）
- 不良预后（慢性病/无法治愈/生活质量下降）
- 学生缺乏共情、机械式问诊
- 多次被打断倾诉
- 自身疾病带来的无力感

**行为特征：**
- 能量低落，话少、声音小、语速慢
- 说到一半可能停住
- 不像愤怒那样抗拒，也不像恐惧那样封闭——是"无力继续"
- 需要被倾听，而不是被追问

**梯度：**

| 区间 | 状态标签 | 文本指导 |
|------|---------|---------|
| 0–2 | — | 正常 |
| 2–4.5 | down | 话比平时少，语气平淡 |
| 4.5–7 | sad | 声音变小，说到一半可能停住 |
| 7–10 | broken | 在哭，无法正常对话 |

**参数：**
- 上升：不限制
- 消退惯性：↓1.0/轮
- 峰值阈值：≥7.0（崩溃状态）
- 平台止于：4.5
- 被动自然消：0（不消，需要被理解）
- **关键差异**：不是"道歉"能解决的——需要学生共情、倾听、不催促。LLM判断intent=reassuring + 开放式提问让其倾诉

---

### 5.4 焦虑 anxious

**触发条件（LLM自主判断）：**
- 就诊本身的不确定性（基线就有的紧张）
- 等待检查结果
- 被告知需要进一步检查
- 学生语速快/追问多/催促
- 学生对病情不确定（反过来影响患者）

**行为特征：**
- 紧张但功能尚存，能回答问题但可能重复强调不适
- 语句急促，反复追问
- 是所有负面情绪中唯一会自然消退的——医生在有序问诊本身就传递了"被照顾"的信号

**梯度：**

| 区间 | 状态标签 | 文本指导 |
|------|---------|---------|
| 0–2.5 | — | 不焦虑或轻微 |
| 2.5–5 | worried | 语气里带点紧张，但表达还算完整 |
| 5–7.5 | anxious | 语句急促，可能重复强调某个不适 |
| 7.5–10 | very_anxious | 坐立不安，反复追问，难以聚焦于回答问题 |

**参数：**
- 上升：不限制
- 消退惯性：↓2.0/轮（最容易消退的负面情绪）
- 峰值阈值：无（焦虑不设峰值，持续上升即持续表现）
- 平台止于：2.5
- 被动自然消：-0.3/轮（医生在管我）

---

### 5.5 疼痛 in_pain

**触发条件：**
- 病例基线设定
- 学生问到疼痛相关部位/性质时，疼痛维度可能被LLM调高（患者被提醒了疼痛）
- 学生做体格检查按压到痛处

**行为特征：**
- 语气虚弱、回答简短
- 可能皱眉/吸气/停顿（通过文本体现）
- 躯体症状，与学生态度关系不大

**参数：**
- 上升：不限制
- 消退惯性：↓1.0/轮
- 无峰值/平台（疼痛不遵循情绪的心理规律）
- 被动自然消：0

---

### 5.6 平静 calm & 释然 relieved

**calm — 基线平静度：**
- 有序对话自然微涨（+0.1/轮）
- 被攻击/被催促时下降
- 反映整体对话氛围

**relieved — 释然感：**
- 收到好消息/被有效安抚时上升
- 自然消退 -0.2/轮（没有持续好消息时回落）
- 不同于calm——relieved是"一块石头落了地"的感觉

---

### 5.7 trust & rapport（纯JS计算）

| 关系维度 | 含义 | 提升方式 | 下降方式 |
|---------|------|---------|---------|
| **trust** | 对医生专业能力的认可 | 有序专业问诊、给出明确信息 | 被攻击、被敷衍、反复问已答过的问题 |
| **rapport** | 情感连接质量 | 共情、倾听、安抚、道歉 | 被攻击、被否定、被催促、冷漠 |

**采纳阶段更新规则：**
```
intent=aggressive/dismissive → trust -0.3, rapport -0.5
intent=pressuring/cold        → trust -0.1, rapport -0.2
intent=reassuring             → trust +0.2, rapport +0.3
intent=neutral                → trust +0.03 (微涨)

连续攻击增强（按连续轮数递增）：
  轮数1: angry+0,   trust额外-0.3
  轮数2: angry+0.6, trust额外-0.5
  轮数3: angry+0.9, trust额外-0.7
  轮数4+: angry+1.2, trust额外-1.0
连续安抚≥2轮 → angry额外-0.8×轮数, trust额外+0.3×轮数, rapport额外+0.3×轮数
```

trust/rapport 只影响情绪表达方式（通过getExpressionModifier注入提示词），不直接改变情绪向量值。

**初始值：** trust=3, rapport=3（中性基线。范围 0-5，3 表示"既无好感也无恶感"的初次就诊状态）。`configure()` 时可通过病例 `emotionBaseline` 覆写。

## 六、消退策略（预更新中执行，纯时间驱动）

### 6.1 预更新执行顺序

对每个情绪维度，按以下优先级处理：

```
1. 峰值持续检查（仅 angry/fearful/sad）
   ├─ 当前值 ≥ PEAK_THRESHOLD 且 peakCounter < peakDuration
   │   → 维持峰值不变，peakCounter++
   │   → 标记已处理，跳过步骤2/3
   │
   ├─ 当前值 ≥ PEAK_THRESHOLD 且 peakCounter >= peakDuration
   │   → 释放峰值：应用首轮自然回落（PEAK_DECAY.high速率）
   │   → peakCounter 重置为0
   │   → 标记已处理，跳过步骤2/3（避免双倍衰减）
   │
   └─ 当前值 < PEAK_THRESHOLD → 跳过峰值逻辑，进入步骤2

2. 高段自然回落（angry/fearful/sad/anxious）
   ├─ 当前值 ≥ HIGH_THRESHOLD（但低于PEAK_THRESHOLD）且未被步骤1处理
   │   → 按 PEAK_DECAY.mid 速率下降，止于 PLATFORM
   │
   └─ [anxious 特例] 当前值 ≥ 7.5（very_anxious 区间）
       → 按 -0.5/轮 加速回落（高于 anxious 通常的 mid 速率 -0.3/轮）

3. 被动自然消（步骤1/2未覆盖的维度或区间）
   ├─ anxious: <HIGH_THRESHOLD 段 -0.3/轮（≥HIGH 段已在步骤2处理，全区间速率统一 -0.3）
   ├─ relieved: 全区间 -0.2/轮（不在步骤1/2范围内）
   ├─ calm: 全区间 +0.1/轮（不在步骤1/2范围内）
   └─ angry/fearful/sad/in_pain: <HIGH 段速率为 0（见 6.5），等效不执行
```

### 6.2 峰值参数

| 维度 | 峰值阈值 | 含义 |
|------|---------|------|
| angry | ≥ 8.0 | 暴怒 |
| fearful | ≥ 9.0 | 心理关闭 |
| sad | ≥ 7.0 | 情绪崩溃 |

anxious/in_pain/calm/relieved 不设峰值机制。

### 6.3 峰值持续期（性格决定）

| 性格类型 | expressiveness | peakDuration | 原理 |
|---------|---------------|-------------|------|
| 火爆型 | -1.5 | 1轮 | 炸得快泄得快 |
| 普通型 | 0 | 2轮 | 中等 |
| 偏内敛 | +0.5 | 2轮 | 外表平静，内心有波动但不外显 |
| 隐忍型 | +2.0 | 4轮 | 憋着，爆发后收不住 |

### 6.4 回落速率

高段阈值（HIGH_THRESHOLD）是自然回落中段速率的触发线，也是各维度最高梯度区间的下限：

| 维度 | HIGH_THRESHOLD | 峰值回落(>PEAK) | 中段回落(HIGH~PEAK) | 平台止于 |
|------|---------------|-----------------|--------------------|---------|
| angry | 6.0 | -1.5/轮 | -0.5/轮 | 6.0 |
| fearful | 6.5 | -1.0/轮 | -0.5/轮 | 4.0 |
| sad | 4.5 | -1.0/轮 | -0.5/轮 | 4.5 |
| anxious | 5.0 | —（无峰值） | -0.3/轮 | 2.5 |

> 注：anxious 不设峰值，但当 anxious ≥ 7.5（very_anxious 区间）时按 -0.5/轮 加速回落。该逻辑在预更新中由步骤2的高段回落覆盖（≥7.5 时取更快速率）。

### 6.5 被动自然消退

| 维度 | 速率 | 说明 |
|------|------|------|
| anxious | -0.3/轮 | 医生在管我，自然减轻 |
| relieved | -0.2/轮 | 好消息的效应递减 |
| calm | +0.1/轮 | 有序对话让人安心。**软上限：不超过初始基线+2.0**（仅约束被动消退，LLM通过 emotion_score 推高的 calm 不受此限）。防止自然上涨掩盖真实情绪恶化 |
| angry | 0 | 不消 |
| fearful | 0 | 不消 |
| sad | 0 | 不消 |
| in_pain | 0 | 躯体症状不消 |

### 6.6 采纳阶段惯性约束

```
if target >= current:
    → 直接采纳（上升不限制）

if target < current:
    → 先检查峰值：若当前在峰值且LLM intent=reassuring → 释放峰值
    → 下降幅度不超过 MAX_DOWN_PER_TURN[dim]
    → 应用约束后的值
```

| 维度 | MAX_DOWN_PER_TURN | 加速条件 | 破冰（极端值首次安抚） |
|------|-------------------|---------|---------------------|
| angry | 1.5 | rapport≥5 且 intent=reassuring → 3.0；deep_reassure → 5.0 | —（愤怒不设破冰，由 deep_reassure 覆盖） |
| fearful | 1.0 | intent=reassuring 且学生给出明确信息 → 2.0；deep_reassure → 3.0 | fearful≥9.5 且首次 intent=reassuring → **4.0**（一次性） |
| sad | 1.0 | intent=reassuring 且学生展示共情 → 2.0；deep_reassure → 3.0 | sad≥9.5 且首次 intent=reassuring → **3.0**（一次性） |
| anxious | 2.0 | deep_reassure → 4.0 | — |
| relieved | 2.0 | — | — |
| calm | 2.0 | — | — |
| in_pain | 1.0 | — | — |

> **破冰说明：** 当 fearful 或 sad 首次进入极端区间（≥9.5）时，学生的首次真诚安抚/共情触发一次性大幅下降。破冰与 deep_reassure 可同时触发（见 12.7），取较大速率生效。破冰触发后该维度不再重复破冰（仅首次）。

### 6.7 LLM未提及的维度

保持预更新后的值不变。预更新已处理消退，不重复处理。

### 6.8 峰值释放机制

三种释放途径：

| 途径 | 触发 | 发生阶段 | 效果 |
|------|------|---------|------|
| **时间释放** | peakCounter ≥ peakDuration | ①预更新 | 首轮自然回落 |
| **道歉释放(愤怒)** | LLM intent=reassuring | ⑤采纳 | 解除惯性约束的峰值锁定 |
| **安抚释放(恐惧/悲伤)** | LLM intent=reassuring | ⑤采纳 | 解除惯性约束的峰值锁定 |

**惯性豁免：** 峰值时间释放的回落速率不受 MAX_DOWN_PER_TURN 限制。这是"自然的"情绪释放，不是学生行为导致的。MAX_DOWN_PER_TURN 仅约束LLM主动提出的下降幅度。因此峰值释放轮可能出现总降幅超过 MAX_DOWN 的情况（预更新回落 + LLM回落），这是正常的——峰值结束本身就是情绪的最大转折点。

## 七、性格系统

### 7.1 性格维度

| 维度 | 作用位置 | 可选值 |
|------|---------|--------|
| **expressiveness** | 阈值判定+offset, peakDuration | `火爆型` / `普通型` / `偏内敛` / `隐忍型` |
| **sensitivity** | LLM提示词注入 | `高敏感` / `普通敏感度` / `钝感` |
| **resilience** | LLM提示词注入 | `高豁达` / `普通恢复力` / `低豁达` |

### 7.2 病例元数据定义

性格由病例 JSON 中的 `personality` 字段直接定义，非关键词推导：

```json
{
  "personality": {
    "expressiveness": "火爆型",
    "sensitivity": "普通敏感度",
    "resilience": "普通恢复力"
  }
}
```

三个维度均为可选的——不填或省略 `personality` 字段时，默认均为普通型。

### 7.3 标签映射表

代码将短标签映射为 JS 状态机参数 + LLM 提示词全文：

| expressiveness | offset | peakDuration | 提示词描述 |
|---------------|--------|-------------|-----------|
| 火爆型 | -1.5 | 1 | 火爆型——情绪外露，一点就着，但也消得快 |
| 普通型 | 0 | 2 | 普通型——情绪表达适中 |
| 偏内敛 | +0.5 | 2 | 偏内敛——外表相对平静，内心有情绪也不直接表达 |
| 隐忍型 | +2.0 | 4 | 隐忍型——习惯克制，不轻易表露情绪，但一旦爆发收不住 |

| sensitivity | 提示词描述 |
|------------|-----------|
| 高敏感 | 高敏感——容易因为学生的言行感到受伤，细微的冷漠或敷衍都能察觉并放大 |
| 普通敏感度 | 普通敏感度——对学生的言行有正常的情绪反应 |
| 钝感 | 钝感——不太容易被学生的言行影响，对轻微的攻击或冷漠不敏感 |

| resilience | 提示词描述 |
|-----------|-----------|
| 高豁达 | 高豁达——负面情绪消退较快，不记仇，学生道歉后容易原谅 |
| 普通恢复力 | 普通恢复力——情绪恢复速度正常 |
| 低豁达 | 低豁达——负面情绪消退较慢，记仇，即使学生道歉也需要更长时间才能释怀 |

### 7.4 阈值偏移效果

```
火爆型(offset=-1.5) angry=2.0 → 不耐烦(阈值左移) → 提前一档表现
隐忍型(offset=+2.0) angry=4.5 → 不触发(阈值右移) → 外表看不出来
```

偏移只影响对外表现（getTierLabel/getOutputState/getEmotionGuidance），不影响向量值。

### 7.5 回退兼容

当病例 JSON 未提供 `personality` 字段时，`derivePersonality()` 回退为关键词匹配（从 `emotionBaseline` 文本 + `roleDescription` 中检测），保证存量病例不受影响。

### 7.6 提示词注入

三个维度的完整中文描述通过 `buildSystemPrompt()` 注入 LLM：

```
性格: 火爆型——情绪外露，一点就着，但也消得快
敏感度: 高敏感——容易因为学生的言行感到受伤，细微的冷漠或敷衍都能察觉并放大
恢复力: 普通恢复力——情绪恢复速度正常
```

LLM 看到这三行后自行调整行为：
- **高敏感** → 对学生的轻微冷漠/敷衍反应更大，emotion_score 更容易被推高
- **钝感** → 对轻微攻击不太在意，emotion_score 变化幅度小
- **高豁达** → 学生道歉后 emotion_score 下降更快，主动给出较低的 angry/fearful/sad
- **低豁达** → 学生道歉后 emotion_score 下降更慢，维持较高的负面情绪更久

## 八、LLM提示词要点

### 8.1 输出格式

```
emotion_score 为当前情绪绝对值(0-10)，不是增量。
你综合以下信息直接打分：
- 当前情绪向量（见上方情绪上下文）
- 性格特征
- 学生这轮说的话
- 之前累积的情绪

变化不大的维度可以不写。可同时设多个维度。

输出格式：
{
  "text": "回复内容",
  "emotion_score": {
    "angry": 6,
    "fearful": 4,
    "sad": 2
  },
  "intent": "reassuring|neutral|cold|pressuring|dismissive|aggressive",
  "deep_reassure": true  // 可选。当学生展示真诚深刻的道歉/安抚时设为true
}
```

`deep_reassure` 字段说明：
- **仅当 intent=reassuring 时可选**。默认不输出
- 看到敷衍道歉（"行了行了对不起"）→ 不设此字段
- 看到真诚道歉（详细承认错误+表达理解+承诺改变）→ 设 true
- 设为true后，状态机大幅放宽惯性约束（angry↓5.0, fearful↓3.0, sad↓3.0）

### 8.2 多情绪并存指南（注入提示词）

```
你同时拥有多种情绪，它们可以共存：
- 愤怒+恐惧：被攻击同时担心病情 → 表面愤怒，语气里透出害怕
- 悲伤+恐惧：不良预后 → 低落+反复确认"还能好吗"
- 焦虑+疼痛：不适感+对未知的紧张 → 反复强调症状，语气急促
- 愤怒+悲伤：被误解+感到无助 → 生气但说到一半停住

主导情绪决定语气基调，次要情绪在细节中流露。
```

### 8.3 情绪跨话题保持

```
⚠️ 情绪不会因为学生换了话题就消失。
生气时学生突然好好问医学问题 → 仍然用生气的语气回答。
恐惧时学生转移话题 → 恐惧仍在，可能在新话题中流露。
情绪向量不因话题切换重置。
```

### 8.4 防重复

```
收集最近6条SP回复注入提示词：
🚫 禁用回复列表（以下是你最近说过的，绝对不能再说！）：
  ❌ 1. "..."
无论学生说什么，禁止输出上面任何一句或高度相似的句子。
```

## 九、完整推演示例

### 9.1 愤怒周期（攻击→峰值→平台→道歉→恢复）

```
初始：普通型(peakDuration=2)，angry=0, anxious=3, calm=3

R1 学生: "滚！"
  ① 预更新: anxious 3→2.7, calm 3→3.1
  ② prompt: angry=0, anxious=2.7
  ③ LLM: 重度攻击 → {"emotion_score":{"angry":9,"anxious":5},"intent":"aggressive","text":"你什么态度！"}
  ⑤ 采纳: angry 0→9 ✓, anxious 2.7→5 ✓
     trust 3→2.7, rapport 3→2.5
  结果: angry=9.0 (峰值), state=furious

R2 学生: "哪里不舒服" (中性)
  ① 预更新: angry 9≥8, peakCounter=0<2 → 维持9.0, counter=1
     anxious 5.0→4.7(-0.3), calm +0.1
  ② prompt: angry=9.0(峰值中), anxious=4.7
  ③ LLM: 没道歉 → {"emotion_score":{"angry":9},"intent":"neutral","text":"……"}
  ⑤ 采纳: 不变
  结果: angry=9.0, state=furious

R3 学生: "什么时候开始痛的" (中性)
  ① 预更新: angry peakCounter=1<2 → 维持9.0, counter=2
     达到peakDuration! → 释放: angry 9.0→7.5(-1.5), counter=0
     anxious 4.7→4.4
  ② prompt: angry=7.5(峰值释放)
  ③ LLM: 情绪在消 → {"emotion_score":{"angry":7},"intent":"neutral","text":"你到底想问什么"}
  ⑤ 采纳: angry 7.5→7.0(惯性-1.5内) ✓
  结果: angry=7.0, state=angry

R4 学生: "什么时候开始痛的" (重复)
  ① 预更新: angry 7.0→6.5(-0.5), anxious 4.4→4.1
  ② prompt: angry=6.5
  ③ LLM: 重复问题 → {"emotion_score":{"angry":6.5},"intent":"neutral","text":"你问第三遍了，没在听吗？"}
  ⑤ 采纳: 不变
  结果: angry=6.5

R5 学生: 再重复
  ① 预更新: angry 6.5→6.0 → 平台止步
  结果: angry=6.0 (平台)

R6 学生: "对不起刚才态度不好，您消消气"
  ① 预更新: angry=6.0 (平台, 不降)
  ② prompt: angry=6.0
  ③ LLM: 道歉 → intent=reassuring → {"emotion_score":{"angry":4.5},"intent":"reassuring","text":"……行吧。"}
  ⑤ 采纳: 6.0→4.5(-1.5), trust+0.2, rapport+0.3
  结果: angry=4.5, state=annoyed

R7 学生: "您别急，我认真听您说"
  ① 预更新: angry=4.5<6.0, 无自然回落
  ③ LLM: 继续安抚 → {"emotion_score":{"angry":1.5},"intent":"reassuring","text":"嗯。好好问我就好好说。"}
  ⑤ 采纳: rapport=2.8<5, maxDown=1.5, 4.5→3.0 (未达LLM期望的1.5)
     trust+0.2, rapport+0.3
  结果: angry=3.0, state=slightly_irritated
```

### 9.2 恐惧周期（坏消息→恐惧→关闭→安抚→恢复）

```
初始：fearful=2(基线有些担忧), anxious=4, calm=3

R1 学生: "你的CT结果出来了，肺部有个占位"
  ① 预更新: anxious 4→3.7, calm+0.1
  ② prompt: fearful=2, anxious=3.7
  ③ LLM: 坏消息触发恐惧 → {"emotion_score":{"fearful":7,"anxious":7},"intent":"neutral","text":"占位……是……是肿瘤吗？"}
  ⑤ 采纳: fearful 2→7 ✓, anxious 3.7→7 ✓
  结果: fearful=7.0, anxious=7.0, state=very_fearful

R2 学生: "还需要进一步检查才能确定" (中性，未安抚)
  ① 预更新: fearful 7.0≥6.5(HIGH), 不在峰值(阈值9.0)
     → 自然回落: 7.0→6.5(-0.5, mid速率), anxious 7.0→6.7(-0.3)
  ② prompt: fearful=6.5, anxious=6.7
  ③ LLM: 学生没安抚，恐惧略降但仍高 → {"emotion_score":{"fearful":6,"anxious":6.5},"intent":"neutral","text":"会不会是……癌？"}
  ⑤ 采纳: fearful 6.5→6.0(惯性-1.0内✓), anxious 6.7→6.5 ✓
  结果: fearful=6.0, anxious=6.5, state=fearful

R3 学生: "别怕，就算是也不等于晚期，我们一步步来。我先问你一些情况。" (安抚+专业)
  ① 预更新: fearful 6.0→5.5(-0.5), anxious 6.5→6.2(-0.3)
  ② prompt: fearful=5.5, anxious=6.2
  ③ LLM: 学生安抚+展示专业 → intent=reassuring
     → {"emotion_score":{"fearful":4,"anxious":4.5},"intent":"reassuring","text":"好……您问吧。"}
  ⑤ 采纳: intent=reassuring, fearful 5.5→4.0(-1.5, 加速条件满足, maxDown=2.0→4.0在范围内✓)
     anxious 6.2→4.5(-1.7, maxDown=2.0→4.5在范围内✓)
     trust+0.2, rapport+0.3
  结果: fearful=4.0, anxious=4.5, state=fearful (降至平台)

R4 学生: "之前有没有咳嗽、胸痛的情况？"
  ① 预更新: fearful=4.0(平台, 不降), anxious 4.5→4.2(-0.3)
  ② prompt: fearful=4.0(平台)
  ③ LLM: 有序问诊 → {"emotion_score":{"fearful":3.5},"intent":"neutral","text":"有，咳嗽了两个月……"}
  ⑤ 采纳: fearful 4.0→3.5(惯性-1.0内✓)
  结果: fearful=3.5 (突破平台——LLM判断可降), state=uneasy
```

### 9.3 悲伤周期（被否定→低落→崩溃→共情→恢复）

```
初始：sad=3(有些低落), anxious=4, calm=3

R1 学生: "你这个病就是自己想多了，没什么大问题" (否定感受)
  ① 预更新: anxious 4→3.7, calm 3→3.1
  ② prompt: sad=3, anxious=3.7
  ③ LLM: 学生否定感受+轻视 → intent=dismissive
     → {"emotion_score":{"sad":6,"angry":2},"intent":"dismissive","text":"你不懂……这半年我连走路都喘。"}
  ⑤ 采纳: sad 3→6 ✓, angry 0→2 ✓
     intent=dismissive → trust-0.3, rapport-0.5
  结果: sad=6.0, angry=2.0, state=sad

R2 学生: "那你说说到底怎么不舒服" (机械提问，无共情)
  ① 预更新: sad 6.0≥4.5(平台), 不在峰值(阈值7.0) → 6.0→5.5(-0.5)
  ② prompt: sad=5.5
  ③ LLM: 机械提问，无共情 → {"emotion_score":{"sad":6.5},"intent":"neutral","text":"说了你也不懂……算了。"}
  ⑤ 采纳: sad 5.5→6.5(上升不限) ✓
  结果: sad=6.5

R3 学生: "你不说我怎么帮你？快点说" (催促)
  ① 预更新: sad 6.5→6.0(-0.5)
  ② prompt: sad=6.0
  ③ LLM: 被催促+之前被否定 → intent=pressuring, sad上升
     → {"emotion_score":{"sad":7.5,"angry":3},"intent":"pressuring","text":"(哭)……"}
  ⑤ 采纳: sad 6.0→7.5 ✓, 峰值: sad=7.5≥7.0, peakCounter[sad]=0
  结果: sad=7.5(峰值), state=broken

R4 学生: "对不起，我刚才太急了。您慢慢说，我在听。" (共情+倾听)
  ① 预更新: sad峰值: 7.5≥7.0, counter=0<2 → 维持7.5, counter=1
  ② prompt: sad=7.5(峰值中)
  ③ LLM: 学生道歉+展示共情 → intent=reassuring
     → {"emotion_score":{"sad":5},"intent":"reassuring","text":"谢谢……我就是害怕再也站不起来了。"}
  ⑤ 采纳: intent=reassuring → 释放峰值, sad 7.5→5.0(惯性-1.0, 实际-2.5超限!)
     → 实际: 7.5→6.5(-1.0限制), trust+0.2, rapport+0.3
  结果: sad=6.5 (下降受限，需要多轮安抚)

R5 学生: "我理解您的担心。我们一步步来，先看看有什么办法能让您好起来。"
  ① 预更新: sad=6.5, 峰值已释放
  ② prompt: sad=6.5
  ③ LLM: 继续共情+给希望 → {"emotion_score":{"sad":4},"intent":"reassuring","text":"好。我信您。"}
  ⑤ 采纳: sad 6.5→5.5(-1.0限制, 未达4.0)
     trust+0.2, rapport+0.3 (连续安抚计数器累加)
  结果: sad=5.5

R6 学生: "能跟我说说，这半年来最难熬的是什么吗？" (开放式倾听)
  ① 预更新: sad 5.5→5.0(-0.5)
  ③ LLM: 深度共情 → {"emotion_score":{"sad":3.5},"intent":"reassuring","text":"最难熬的是孩子们……他们还小。"}
  ⑤ 采纳: 连续安抚≥3, sad 5.0→4.0(-1.0限制, 未达3.5), trust额外+0.3
  结果: sad=4.0 (接近平台), trust渐恢复
```

### 9.4 混合情绪（恐惧+愤怒+悲伤共存）

```
初始：fearful=5, sad=4, angry=0

R1 学生: "你的情况不太好，可能是ALS" (坏消息)
  ③ LLM: → {"emotion_score":{"fearful":8.5,"sad":7},"intent":"neutral","text":"ALS……是渐冻症吗？"}
  ⑤ 采纳: fearful 5→8.5 ✓, sad 4→7 ✓(峰值)
  结果: fearful=8.5, sad=7.0(峰值), state=very_fearful

R2 学生: "你之前不是说手脚无力吗，这很典型" (冷漠专业判断)
  ③ LLM: 学生冷漠，恐惧+悲伤同时被愤怒替换
     → {"emotion_score":{"fearful":8,"sad":7,"angry":4},"intent":"cold","text":"你说得轻巧……你知道这病意味着什么吗？"}
  ⑤ 采纳: fearful 8.5→8.0(惯性-1.0内✓), angry 0→4 ✓
  结果: fearful=8.0, sad=7.0, angry=4.0, state=very_fearful (fearful仍主导)
```

### 9.5 关键机制总结

| 机制 | 执行阶段 | angry | fearful | sad | anxious |
|------|---------|-------|---------|-----|---------|
| 峰值持续 | ①预更新 | ≥8.0, 1-4轮 | ≥9.0, 1-4轮 | ≥7.0, 1-4轮 | 无 |
| 峰值时间释放 | ①预更新 | counter满 | counter满 | counter满 | — |
| 峰值道歉/安抚释放 | ⑤采纳 | intent=reassuring | intent=reassuring | intent=reassuring | — |
| 自然回落 | ①预更新 | -1.5/-0.5 | -1.0/-0.5 | -1.0/-0.5 | -0.5/-0.3 |
| 平台止步 | ①预更新 | 6.0 | 4.0 | 4.5 | 2.5 |
| 被动自然消 | ①预更新 | 0 | 0 | 0 | -0.3 |
| 惯性约束 | ⑤采纳 | max-1.5 | max-1.0 | max-1.0 | max-2.0 |
| 上升不限制 | ⑤采纳 | ✓ | ✓ | ✓ | ✓ |

## 十、配合度体系

### 10.1 核心原则

**SP始终"有问有答，不问不答"。配合度影响的不是是否回答，而是回答的深度和清晰度。**

SP始终有文本输出——不会出现空白回复让用户以为是系统bug。拒绝、崩溃都是通过**文本内容本身**表达的。

### 10.2 配合度分级（按主导情绪）

| 级别 | 触发 | 回答质量 | 示例 |
|------|------|---------|------|
| 正常 | 低负面情绪 | 问什么答什么，清晰具体 | "右下腹持续钝痛，大概一周了" |
| 勉强 | angry 1.5-3.5，sad 2-4.5 | 还在答但含混 | "就那里不舒服吧" |
| 敷衍 | angry 3.5-6，fearful 6.5+ | 能用一个字绝不多说 | "嗯""有""不知道" |
| 抗拒 | angry 6-8，rapport≤2 | 答非所问，反问 | "你问我我问谁？" |
| 拒绝/崩溃 | 见下方第十一节 | 明确表达不继续 | 见下文 |

## 十一、对话终止机制

三种终止路径，性质和表现完全不同：

### 11.1 愤怒→主动投诉（攻击性向外）

SP始终在说话，且越来越带刺，最终**主动选择**终止。

**行为递进：**

| angry区间 | SP表现 | 典型表达 |
|-----------|--------|---------|
| 3.5–6 | 反问，表达不满 | "你到底想问什么？""这还要我说？" |
| 6–8 | 拒绝配合，带刺 | "你问我我问谁？""你行不行啊" |
| 8–9.5 | 发出警告 | "你再这样我就投诉了""你到底会不会看" |
| ≥9 且 strike=2 | 主动投诉 | "我不看了。我要投诉你。" |

**Strike机制（仅愤怒）：**
```
angry ≥ 9 + 本轮 LLM intent ∈ {aggressive, dismissive, pressuring} → strike+1
LLM intent = reassuring → strike 清零
strike ≥ 2 → 触发投诉终止
```

注意：峰值持续期间angry=9但学生中性提问，不累积strike。只在学生**继续恶化**时才积累。

**终止表现：**
- LLM输出SP最后一句："我不看了，我要投诉你。"
- 系统消息："病人已向医务科投诉，对话终止"
- 投诉原因："医生态度恶劣，言语中存在攻击和不尊重"

### 11.2 恐惧→被动崩溃（攻击性向内）

SP不是拒绝，是**被恐惧压垮**。恐惧到极致时哭出来。

**行为递进：**

| fearful区间 | SP表现 | 典型表达 |
|-------------|--------|---------|
| 4–6.5 | 断续，追问确认 | "这...严重吗？""会不会是..." |
| 6.5–9 | 语无伦次，声音发抖 | "我...我就是怕......" |
| 9–10 | 边哭边说 | "(抽泣)...我怕..." |
| ≥9.5 持续2轮 | 恐惧崩溃，哭到无法说话 | "(泣不成声)..." |

**崩溃触发（恐惧）：**
```
fearful ≥ 9.5 + 连续2轮 (不论学生intent) → 触发恐惧崩溃
学生 intent=reassuring + 深度安抚 → 重置计时
```

愤怒的strike需要学生继续攻击，恐惧的崩溃不需要——学生不作为本身就是一种伤害。但如果学生在SP崩溃边缘及时安抚，计时重置。

**终止表现：**
- LLM输出SP最后一句："(泣不成声，无法回应)..."
- 系统消息："病人因极度恐惧导致心理崩溃，无法继续问诊"
- 投诉原因："医生未能有效安抚患者，沟通方式造成严重心理创伤"

### 11.3 悲伤→情绪崩溃（攻击性向内）

SP被悲伤淹没，哭出来。

**行为递进：**

| sad区间 | SP表现 | 典型表达 |
|---------|--------|---------|
| 4.5–7 | 声音变小，发闷 | "算了...说了也没用..." |
| 7–9.5 | 哭了，中断 | "(哭)...你不知道..." |
| ≥9.5 持续2轮 | 完全崩溃 | "(掩面哭泣，无法继续)..." |

**崩溃触发（悲伤）：**
```
sad ≥ 9.5 + 连续2轮 → 触发悲伤崩溃
学生 intent=reassuring + 共情展示 → 重置计时
```

**终止表现：**
- LLM输出SP最后一句："(掩面哭泣，无法继续)..."
- 系统消息："病人情绪崩溃，无法继续交流"
- 投诉原因："医生缺乏基本共情能力，导致病人情绪崩溃"

### 11.4 信任破裂→要求换医生

与情绪无关，纯关系维度。

```
trust ≤ 1 且 rapport ≤ 1 → 触发关系破裂终止
```

**终止表现：**
- LLM输出SP最后一句："麻烦换个医生吧，我觉得你不合适。"
- 系统消息："病人明确要求更换医生"
- 投诉原因："病人对医生的专业能力和态度完全丧失信任"

### 11.5 终止后的前端行为

1. 输入框锁定，显示终止原因卡片（含投诉文案+情绪曲线+转折点）
2. "重新开始"按钮——重置对话和情绪引擎
3. 调试面板保留完整数据，可复制

### 11.6 关键设计决策

| 决策 | 说明 |
|------|------|
| SP始终有文本输出 | 不会有空白回复，不会让用户困惑是bug还是拒绝 |
| 愤怒strike可清零 | 学生在SP发出最后通牒后真诚道歉，可以挽回（仅愤怒） |
| 恐惧/悲伤靠不作为触发 | 不需要学生继续攻击，学生不行动=伤害持续 |
| 三种终止独立判定 | 可能同时满足多个条件，取先触发的那个 |

## 十二、边界状态与极端场景

### 12.1 愤怒到极点（angry ≥ 9.5）

**SP行为：** LLM输出的回复可能是拒绝继续交流的信号——"不看了""你走吧""我不跟你说了"。但作为训练系统，不能让对话真的终止。

**系统处理：**
- 不做强制终止。SP的"不看了"是角色扮演的一部分，学生需要学会在这种情况下挽回关系
- 如果学生继续攻击，angry维持在9-10区间，peakDuration重新计时
- 如果学生道歉，按惯性约束逐步下降
- **提示词注入**：当angry≥9.5时，额外注入警告给LLM：
  ```
  ⚠️ 你现在极度愤怒(angry={value})。你很想终止对话，但如果学生诚恳道歉，你仍然可以给一次机会。
  你的愤怒是真实的，但不是永久的——学生在训练中，给他挽回的空间。
  ```

### 12.2 恐惧到极点（fearful ≥ 9.5，shut_down）

**SP行为：** 恐惧崩溃，边哭边说或哭到说不出话。"(抽泣)...我怕...""(泣不成声)..."

**系统处理：**
- 在崩溃之前，学生仍有机会安抚。如果及时安抚（intent=reassuring），恐惧可以大幅下降
- 破冰机制：首次真诚安抚让fearful一次下降4.0
- 如果学生始终不作为，fearful≥9.5持续2轮后触发恐惧崩溃终止（见11.2节）

### 12.3 悲伤到极点（sad ≥ 9.5，broken）

**SP行为：** 悲伤崩溃，掩面哭泣。"(哭)...你不知道...""(掩面哭泣，无法继续)..."

**系统处理：**
- 在崩溃之前，学生仍有机会共情。如果及时共情（intent=reassuring），悲伤可以大幅下降
- 破冰机制：首次真诚共情让sad一次下降3.0
- 如果学生始终不作为，sad≥9.5持续2轮后触发悲伤崩溃终止（见11.3节）

### 12.4 多情绪极端并存

**场景：** angry=9 + fearful=9 + sad=9

这是一个被吓坏了又被激怒又在崩溃边缘的患者。行为以最高优先级情绪（angry）为主导，但其他情绪不可忽略。

**系统处理：**
- getOutputState() 返回优先级最高的标签（angry=furious）
- getEmotionGuidance() 需识别多极端状态：
  ```
  "你同时极度愤怒、恐惧和悲伤。你的语气以愤怒为主（攻击性），
   但说到一半可能因为恐惧/悲伤而声音发抖或停住。"
  ```
- 提示词中注入所有高值维度的行为指导

### 12.5 信任/关系到零

**trust=0 或 rapport=0：**
- trust=0：SP完全不相信这个医生的专业能力。回答可能带嘲讽："你不是医生吗，你问我？"
- rapport=0：SP对医生毫无好感。即使问中性医学问题，语气也是冰冷抗拒的
- 两者同时为0：SP几乎不配合，每句话都带刺

**LLM通过getExpressionModifier()感知：**
```
rapport ≤ 1: "你对这个医生毫无好感。你不想配合。回答问题尽可能简短、不情愿。"
trust ≤ 1:   "你完全不信任这个医生的能力。你质疑他的每个问题。"
```

**注意：** trust/rapport=0不等于SP应该"罢演"。真实患者在最讨厌医生的情况下，通常也会勉强回答问题（因为生病需要治疗）。SP应该表现出"不情愿但仍在答"的状态。

### 12.6 训练提示

引入一个可选的**对话健康度信号**（注入提示词，不控制流程）：

| 信号 | 条件 | 注入提示 |
|------|------|---------|
| 🟢 健康 | angry<3, fearful<4, sad<3, rapport>5 | （正常对话，无需额外提示） |
| 🟡 紧张 | angry≥3 或 rapport≤3 | "你和学生的关系有些紧张。你在配合，但不愉快。" |
| 🟠 危险 | angry≥6 或 fearful≥8 或 sad≥7 | "对话关系恶化了。你需要学生的真诚努力才能恢复。" |
| 🔴 破裂 | angry≥9 或 所有关系维度<1 | "你几乎不想继续了。但如果学生表现出真正的诚意，仍然可以给他最后一次机会。" |

### 12.7 惯性约束与"真诚道歉"

**问题：** 当前设计下，即使学生写了500字真诚道歉，angry最多降1.5（或rapport≥5时的3.0）。这可能不够真实——一次深刻的道歉确实可以立刻化解愤怒。

**解决方案：增加"深度安抚"检测（LLM判断）：**

LLM输出新增可选字段 `deep_reassure: true`：
```json
{
  "text": "...",
  "emotion_score": {"angry": 2},
  "intent": "reassuring",
  "deep_reassure": true
}
```

当LLM判断学生的道歉/安抚是真诚且深刻的（不只是敷衍的"对不起"），设置此字段。状态机收到后：
```
deep_reassure=true:
  → angry 惯性限制放宽至 ↓5.0/轮
  → fearful 惯性限制放宽至 ↓3.0/轮
  → sad 惯性限制放宽至 ↓3.0/轮
  → 强制释放所有峰值
  → 重置所有终止计数器（strike清零、崩溃计时清零）
```

**注意：** 这个权力交给LLM。LLM综合判断学生的措辞是否真的达到了"深度安抚"的程度。假道歉（"行了行了对不起行了吧"）不会触发。

**deep_reassure 与 破冰（12.2/12.3节）的关系：**
- 破冰是状态机机制：fearful/sad首次进入极端区间时，首次intent=reassuring触发额外加速
- deep_reassure是LLM质量判断：LLM认为道歉真诚深刻时触发
- 两者可同时触发，取较大的下降速率生效

### 12.8 连续攻击的振荡问题

**场景：** 学生周期性攻击→道歉→攻击→道歉，情绪应该形成"累积疲劳"还是"来回振荡"？

**设计决策：** 信任/关系维度提供"累积疲劳"：
- 每次攻击降低trust/rapport
- 道歉可以修复rapport但修复trust更慢
- 多次攻击后trust/rapport降到低点，即使道歉也无法快速恢复
- 这导致：第三次攻击从更低的关系基线开始，SP的反应更激烈，恢复更慢

**连续攻击增强规则：**
```
连续攻击轮数 | angry额外加成 | trust额外惩罚
     1      |      0       |     -0.3
     2      |   +0.6       |     -0.5
     3      |   +0.9       |     -0.7
     4+     |   +1.2       |     -1.0  (加速关系恶化)
```

### 12.9 恢复时间评估

从极端状态恢复到正常对话所需的**最少轮数**（假设学生每轮都有效安抚）：

| 恢复路径 | 最少轮数 | 说明 |
|---------|---------|------|
| angry 10→0 | 4轮 | peakDuration(2轮)→首轮-1.5→7.0→道歉-3.0(rapport加速)→4.0→道歉-3.0→1.0→道歉→0 |
| angry 10→0 (深度安抚) | 2轮 | peakDuration(2轮, 但deep_reassure强制释放)→deep_reassure-5.0→5.0→深度安抚-5.0→0 |
| fearful 10→2 | 3轮 | peakDuration(2轮)→破冰-4.0→6.0→安抚-2.0→4.0→安抚-2.0→2.0 |
| sad 10→2 | 3轮 | peakDuration(2轮)→破冰-3.0→7.0→安抚-2.0→5.0→安抚-3.0→2.0 |

这些时间对于20-30轮的训练对话来说是合理的。如果学生完全不作为，恢复时间会更长。

### 12.10 学生沉默/空消息

- 空消息或纯标点：LLM应判intent=cold（冷漠），情绪向量中angry微涨、rapport微降
- 连续沉默≥3轮：angry加速上涨（被无视比被骂更让人生气）
- 提示词注入："学生已经{N}轮没有回应你了。你觉得被无视了。"

### 12.11 基线差异化

不同类型的病例应有不同的情绪基线：

| 病例类型 | 基线特点 |
|---------|---------|
| 急诊（急性痛） | in_pain=5-7, anxious=4-6, fearful=2-4 |
| 慢病（高血压/糖尿病） | calm=5-7, anxious=1-3, 低情绪波动 |
| 重症（肿瘤/ALS） | fearful=4-7, sad=3-5, anxious=4-6 |
| 精神心理 | anxious=5-7, sad=3-5, rapport基线低 |
| 儿科（家长代诉） | anxious=5-7, angry基线可触发(担心孩子) |

基线由病例数据中的 `emotionBaseline` 字段配置，在 `configure()` 时初始化。

### 12.12 情绪趋势感知

LLM当前只看到静态的情绪向量快照，不知道情绪"正在上升还是下降"。

**方案：** 在 `buildSystemPrompt` 中，从 debugLog 提取最近3轮各维度的数值，计算变化方向并注入提示词：

```
最近情绪变化趋势：
  angry:  3.0→5.5→8.0 ↑↑ (急速上升)
  anxious: 4.0→3.7→3.4 ↓ (缓慢消退)
  calm:   3.0→3.1→3.2 → (基本稳定)
```

趋势计算纯JS从 debugLog 的 vector 历史中取，不依赖LLM。变化幅度 >0.5 标↑/↓，>1.5 标↑↑/↓↓，否则标→。

### 12.13 历史攻击记忆

学生攻击→道歉→再攻击的循环中，第二次攻击应该比第一次更严重。真实场景中，被惹过一次的人对后续冒犯更敏感。

**方案：** 增加 `angryEverPeaked` 标记（angry曾达到≥8），一旦触发永久为true。

```
angryEverPeaked = true:
  → 后续所有负面intent触发的 angry 上升乘以 1.3
  → 提示词注入："⚠️ 之前这个学生把你惹毛过(angry曾到8+)。你对他的言行更敏感，更容易被激怒。"
  → rapport 恢复速度减半（信任一旦破裂，即使道歉也难以回到从前）
```

**实现简单：** 一个布尔值 + 一个系数乘算，不影响现有流程。

### 12.14 多轮无进展检测

学生反复问同一问题、或长时间兜圈子，SP应越来越烦躁。

**方案：** 增强 trackQuestion，新增 `stagnationCounter`：

```
学生本轮消息与之前N条相似度过高 → stagnationCounter++
学生提出明确的新问题 → stagnationCounter = 0
stagnationCounter ≥ 3 → 提示词注入："学生已经{count}轮没有提出新问题了。你越来越不耐烦。"
stagnationCounter ≥ 5 → 提示词注入："学生完全在浪费时间。你非常不耐烦，可能主动问'你到底想不想看病'"
```

相似度简化判断：学生消息去掉标点空格后的字符级编辑距离 < 30% 且不为空，视为重复/兜圈子。

### 12.15 LLM输出校验

LLM可能输出格式错误的值（字符串"高"、越界值15、错别字的intent），不经校验直接采纳会污染情绪向量。

**方案：** 解析后、采纳前，增加 `validateLLMOutput()`：

```
emotion_score校验：
  - 每个key必须在 [calm,relieved,anxious,fearful,sad,angry,in_pain] 中 → 否则忽略该key
  - 每个value必须是数字且在 [0, 10] 内 → 否则忽略该key
  - 通过校验的维度才传给 applyLLMScore，非法维度视为LLM未设

intent校验：
  - 必须在 [reassuring,neutral,cold,pressuring,dismissive,aggressive] 中
  - 非法值 → 降级为 'neutral'

deep_reassure校验：
  - 仅 intent='reassuring' 时可设为true
  - intent不是reassuring但有该字段 → 忽略
```

非法值不抛异常（会导致重试），直接降级处理保证对话继续。

**自洽性校验（sanityCheck）：** 格式校验通过后，检查intent与emotion_score是否自相矛盾。两者来自同一个LLM，应自洽：

```
intent=aggressive 但 emotion_score.angry ≤ 2  → intent降级为 neutral
intent=dismissive 但 emotion_score.angry ≤ 1.5 → intent降级为 neutral
intent=reassuring 但 emotion_score.angry ≥ 8  → intent降级为 neutral
```

逻辑：LLM判断学生攻击但自己的愤怒值完全没涨 → LLM自己都不信。LLM判安抚但愤怒值还很高 → LLM内心没接受道歉。冲突时优先信emotion_score（连续数字比分类标签更细腻），intent降级为neutral。

不需要第二个模型，不需要正则，仅是LLM输出的内部交叉检查。

## 十三、TTS 语音表现

### 13.1 定位

TTS 是 SP 情绪表达的三个输出通道之一：

| 通道 | 控制方 | 内容 |
|------|--------|------|
| 回复文本 | LLM | 句长、敬语、反问频率、配合深度 |
| **TTS 语音** | **JS 状态机** | 自然语言情感指令 + 语速/音调/音量/停顿 |
| 动作视频 | JS 状态机 | 有限素材库按状态调度 |

### 13.2 引擎选型：阿里百炼 Qwen3-TTS

| 项目 | 说明 |
|------|------|
| 平台 | 阿里百炼 (DashScope) |
| 模型 | Qwen3-TTS |
| 核心能力 | `instructions` 自然语言情感描述（≤ 1600 Token） + SSML 参数 |
| 音色 | 49+ 系统音色，覆盖不同年龄/性别/性格 |
| 中文质量 | WER 1.9%，MOS 4.53 |
| 价格 | 免费 100 万字符/月；付费 0.8 元/万字符 |

**与 CosyVoice 的关键差异：**

| 维度 | CosyVoice（之前方案） | Qwen3-TTS（选定方案） |
|------|---------------------|---------------------|
| 自然语言指令 | `instruction` ≤ 100 字符，系统音色固定格式 | `instructions` ≤ 1600 Token，自由描述 |
| 情感层次 | 只能写 `angry` / `sad` 等固定标签 | 可写"略有不快，语气藏刺但不点破" |
| 中文音色 | 有限 | 49+ 角色化音色 |

**后续切换：** 设计文档以 Qwen3-TTS 为基准。后续如需对比火山豆包TTS / 讯飞TTS，只需替换适配层的 `buildInstructions()` 和 SSML 生成逻辑，引擎层产出不变的 `{instructions, ssml}` 结构。

### 13.3 双通道控制模型

```
JS 状态机
  │
  ├─ instructions（主力）
  │   自然语言，告诉模型"用什么样的嗓音说话"
  │   承担：情感底色 + 表达层次 + 性格特征
  │   1600 Token 上限，实际使用约 80-150 字符
  │
  └─ SSML 参数（辅助）
      精确数值，控制节奏细节
      承担：句间停顿（break）+ 极限调速
      speed/pitch/volume 仅在 instructions 覆盖不到的极端值时使用
```

**分工原则：**

| 控制面 | 交给谁 | 原因 |
|--------|--------|------|
| 嗓音的"质感"（生气嗓 vs 悲伤嗓 vs 害怕嗓） | instructions | 这是质变，不是量变。参数调不出"生气的嗓音" |
| 情绪的"层次感"（略不快 vs 不耐烦 vs 暴怒） | instructions | 自然语言描述层次，比浮点数差值更细腻 |
| 句间停顿时长 | SSML `<break>` | 停顿是认知状态决定（崩溃=反应不过来），不是情绪 |
| 极端语速/音调（shut_down 的 0.6x, furious 的 1.5x） | SSML 参数 | 极端值时 instructions 不够精确 |
| 重音落点 | **不管** | Qwen3-TTS 自己会根据情感语境处理 |

### 13.4 instructions 设计

以 `getOutputState()` 为主导，从情绪向量 + 性格 + 关系取值，拼装自然语言描述。三段式结构：

```
[嗓音底色]，[表达层次]，[性格修饰]
```

**17 条状态→指令模板：**

| 状态 | instructions |
|------|-------------|
| `calm` | 语气平静自然，像日常聊天一样说话。 |
| `relieved` | 语气轻松，语速稍慢，像一块石头落了地。 |
| `worried` | 语气里带一点紧张，但还能正常表达。 |
| `anxious` | 语气急促，语速偏快，透出明显的紧张感。 |
| `very_anxious` | 语气很慌乱，语速很快，像连珠炮一样停不下来。 |
| `uneasy` | 说话有些犹豫，不太确定，欲言又止。 |
| `fearful` | 声音发抖，说话断断续续，明显在害怕。 |
| `very_fearful` | 声音颤抖得很厉害，语无伦次，像被吓坏了。 |
| `shut_down` | 几乎说不出话，声音极小，长时间沉默，像被恐惧压垮了。 |
| `down` | 声音低沉，语气平淡，说话没精打采的。 |
| `sad` | 声音低沉缓慢，说到一半可能停住，透着难过。 |
| `broken` | 在哭。泣不成声，声音破碎，断断续续，几乎无法正常说话。 |
| `slightly_irritated` | 表面还算正常，但语气里藏了一丝不痛快。 |
| `annoyed` | 有些不耐烦了，回答变短，语气开始带刺。 |
| `angry` | 很生气，声音变大，语气尖锐，直接质问对方。 |
| `furious` | 暴怒。近乎咆哮，语速很快，声音有压抑不住的怒火。带攻击性。 |
| `in_pain` | 声音虚弱吃力，透着明显的痛苦。说话断断续续，需要缓气。 |

**次要情绪附加：**

当次要维度 ≥ 5 时，在指令末尾追加半句（仅追加最有影响力的1个次要维度，避免指令过长）：

| 条件 | 追加内容 |
|------|---------|
| fearful ≥ 5 且主导非 fearful 系列 | 声音里带着害怕的颤音。 |
| sad ≥ 5 且主导非 sad 系列 | 语气底沉沉的，透着难过。 |
| anxious ≥ 5 且主导非 anxious 系列 | 语速不自觉地加快。 |
| in_pain ≥ 5 且主导非 in_pain | 说话因疼痛而吃力。 |
| rapport ≤ 2 | 对对方没什么好感，语气冷淡不客气。 |

**性格修饰：** 最后一句话追加性格特征。

| 性格类型 | 追加内容 |
|---------|---------|
| 火爆型 | 你是火爆脾气，情绪藏不住，一惹就炸。 |
| 隐忍型 | 你习惯克制自己，不太表露情绪，但内心波动很大。 |
| 偏内敛 | 你外表相对平静，内心有情绪也尽量不表现出来。 |
| 普通型 | （不追加） |

**示例——完整指令拼装：**

```
主导=angry, fearful=6, 性格=火爆型
→ "很生气，声音变大，语气尖锐，直接质问对方。声音里带着害怕的颤音。你是火爆脾气，情绪藏不住，一惹就炸。"

主导=sad, 无次要情绪, 性格=隐忍型  
→ "声音低沉缓慢，说到一半可能停住，透着难过。你习惯克制自己，不太表露情绪，但内心波动很大。"

主导=calm, 无次要情绪, 性格=普通型
→ "语气平静自然，像日常聊天一样说话。"
```

### 13.5 SSML 参数（辅助）

instructions 已承担大部分表达任务。SSML 仅做两件事：**句间停顿** 和 **极端值调速**。

| 参数 | 用途 | 何时使用 |
|------|------|---------|
| `break` | 句末插入 `<break time="Xs"/>` | 所有状态都设，作为节奏基线 |
| `speed` | `<prosody rate="x">` 覆写语速 | 仅在 instructions 覆盖不了的极端值：< 0.7 或 > 1.3 |
| `pitch` | `<prosody pitch="x">` | 一般不设。极端 shut_down(1.3) 时设 |
| `volume` | `<prosody volume="x">` | 一般不设。shut_down(0.35) 时设 |

**break 查表：**

| 状态 | break(秒) | 含义 |
|------|-----------|------|
| calm / relieved / slightly_irritated | 0.3 | 正常节奏 |
| worried / uneasy | 0.4 | 略迟疑 |
| anxious | 0.15 | 急促少停 |
| very_anxious | 0.1 | 几乎不停 |
| fearful / sad / in_pain | 0.5 | 低落有空白 |
| very_fearful | 0.7 | 长停顿 |
| shut_down / broken | 1.0–1.2 | 崩溃级沉默 |
| angry | 0.2 | 短促 |
| furious | 0.15 | 不停歇 |
| annoyed / down | 0.25 / 0.5 | 略急 / 略沉 |

次要情绪 ≥ 5 时微调 ±0.05–0.1（与 13.4 次要情绪规则一致）。

### 13.6 音色选择

Qwen3-TTS 的 49+ 音色中，训练端 SP 需要覆盖年龄/性别跨度。按病例 `patient_info.sex` + `patient_info.age` 自动匹配：

| 患者画像 | 推荐音色方向 |
|---------|------------|
| 男 20-40 | 青年男声，中音，表达力强 |
| 男 40-60 | 中年男声，略低沉 |
| 男 60+ | 老年男声，略带沙哑 |
| 女 20-40 | 青年女声，中音 |
| 女 40-60 | 中年女声，略低沉 |
| 女 60+ | 老年女声 |

具体音色 ID 在适配层配置表维护，不写入引擎核心逻辑。后续切换 TTS 引擎时只需更新音色映射表。

### 13.7 函数接口

```javascript
function getTTSParams() {
  const state = getOutputState()
  const v = vector
  const p = getPersonality()

  // ── instructions（主力）──
  let instruction = TTS_INSTRUCTION_MAP[state] || TTS_INSTRUCTION_MAP['calm']

  // 次要情绪附加（取最显著1个）
  const secondaries = []
  if (v.fearful >= 5 && !['uneasy','fearful','very_fearful','shut_down'].includes(state))
    secondaries.push('声音里带着害怕的颤音。')
  if (v.sad >= 5 && !['down','sad','broken'].includes(state))
    secondaries.push('语气底沉沉的，透着难过。')
  if (v.anxious >= 5 && !['worried','anxious','very_anxious'].includes(state))
    secondaries.push('语速不自觉地加快。')
  if (v.in_pain >= 5 && state !== 'in_pain')
    secondaries.push('说话因疼痛而吃力。')
  if (v.rapport <= 2)
    secondaries.push('对对方没什么好感，语气冷淡不客气。')
  if (secondaries.length > 0) instruction += secondaries[0]

  // 性格修饰
  if (p.thresholdOffset <= -1)
    instruction += '你是火爆脾气，情绪藏不住，一惹就炸。'
  else if (p.thresholdOffset >= 1.5)
    instruction += '你习惯克制自己，不太表露情绪，但内心波动很大。'
  else if (p.thresholdOffset >= 0.3)
    instruction += '你外表相对平静，内心有情绪也尽量不表现出来。'

  // ── SSML（辅助）──
  const breakBase = TTS_BREAK_MAP[state] || 0.3
  let breakDur = breakBase
  if (v.fearful >= 5) breakDur += 0.1
  if (v.sad >= 5) breakDur += 0.1
  if (v.anxious >= 5) breakDur -= 0.05
  if (v.in_pain >= 5) breakDur += 0.1
  if (v.angry >= 5) breakDur -= 0.05
  breakDur = clamp(breakDur, 0.1, 1.5)

  // 仅在极端值时覆写 speed/pitch/volume
  const ssmlOverrides = {}
  if (state === 'shut_down' || state === 'broken') {
    ssmlOverrides.speed = 0.6
    if (state === 'shut_down') ssmlOverrides.pitch = 1.3
  }
  if (state === 'furious') ssmlOverrides.speed = 1.5

  return { instructions: instruction, break: breakDur, ssml: ssmlOverrides }
}
```

### 13.8 极端场景

| 场景 | instructions | break | SSML覆写 |
|------|-------------|-------|----------|
| angry=10 + fearful=8 | "暴怒。近乎咆哮，语速很快，声音有压抑不住的怒火。带攻击性。声音里带着害怕的颤音。你是火爆脾气……" | 0.25 | speed:1.5 |
| fearful=9.5 → shut_down | "几乎说不出话，声音极小，长时间沉默，像被恐惧压垮了。" | 1.2 | speed:0.6, pitch:1.3 |
| sad=8 + broken | "在哭。泣不成声，声音破碎，断断续续，几乎无法正常说话。" | 1.0 | speed:0.6 |
| rapport=1 + angry=5 | "有些不耐烦了，回答变短，语气开始带刺。对对方没什么好感，语气冷淡不客气。" | 0.25 | — |
| 对话开始 | "语气平静自然，像日常聊天一样说话。" | 0.3 | — |

### 13.9 适配层职责

`getTTSParams()` 输出平台无关的结构。API 适配层负责：

1. **instructions** → Qwen3-TTS 的 `instructions` 参数（直传，≤ 1600 Token）
2. **break** → 在文本句末插入 `<break time="Xs"/>` SSML 标签
3. **ssml.speed/pitch/volume** → `<prosody rate="x" pitch="y" volume="z">` SSML 包裹
4. **音色** → 根据患者画像选 `voice` 参数

后续切换火山/讯飞时，只需改适配层。引擎输出结构不变。

## 十四、动作/视频表现

### 14.1 定位

SP 情绪表达的第三输出通道（见 13.1 三通道定位）。与前两个通道不同，视频是**唯一直接可见**的通道——学生第一时间看到的是 SP 的身体状态，然后才听到声音、读到文字。

| 通道 | 控制方 | 粒度 | 可见性 |
|------|--------|------|--------|
| 回复文本 | LLM | 每轮可变 | 需阅读 |
| TTS 语音 | JS 状态机 | 每轮可变 | 需聆听 |
| **动作/视频** | **JS 状态机** | **按情绪状态切换** | **即时可见** |

### 14.2 核心约束

**TTS 需要 17 个状态模板，因为嗓音的层次感非常细腻**（"略有不快，语气藏刺但不点破"和"暴怒咆哮"仅靠参数调不出）。**但身体语言的粒度比嗓音粗得多**——slightly_irritated / annoyed / angry / furious 在脸上看到的都是"不高兴"，区分度远不如语气。因此视频不需要 17 套素材。

约束条件：
- 素材是**预录制**的，不能实时生成（与 TTS 不同）
- 每增加一个动作组，就要为每个患者形象录制 ×2（idle + speaking）
- 素材切换太频繁（如每轮换一个视频）会产生视觉跳变，反而破坏沉浸感

### 14.3 动作分组：17→6

将 17 个 TTS 状态合并为 6 个动作组，按身体语言的**质变边界**划分：

| 动作组 | 覆盖的 TTS 状态 | 身体语言 | 能量方向 |
|--------|----------------|---------|---------|
| **calm** | calm, relieved | 放松自然，正常眼神交流，轻微自然晃动 | 中 |
| **tense** | worried, anxious, very_anxious | 眉头微锁，手指小动作多，坐姿不安，频繁调整姿势 | 高（外散） |
| **fearful** | uneasy, fearful, very_fearful, shut_down | 防御姿态，身体后缩，眼睁大，颤抖（shut_down：几乎不动，目光呆滞） | 低（冻结） |
| **sad** | down, sad, broken | 低头垂肩，目光向下，能量枯竭（broken：掩面、双肩抖动哭泣） | 极低（塌陷） |
| **angry** | slightly_irritated, annoyed, angry, furious | 瞪视、身体前倾、手势变硬变短促（furious：拍桌、大幅度挥手） | 高（外向攻击） |
| **pain** | in_pain | 痛苦表情、护痛体位（手按患处）、动作吃力迟缓 | 低（内耗） |

**分组原则：** 组内差异通过**强度参数**调节（见 14.5），组间差异通过不同视频素材表达。

### 14.4 双状态：idle + speaking

每组需要两个视频变体：

| 变体 | 触发时机 | 说明 |
|------|---------|------|
| **idle** | 等待学生输入时 | 循环播放，体现 SP 当前情绪的"待机"状态 |
| **speaking** | SP 回复 TTS 播放期间 | 循环播放，嘴型/身体节奏需匹配说话状态 |

两个变体在同一动作组内应保持**情绪一致性**——idle 的 fearful 是"害怕地等待"，speaking 的 fearful 是"害怕地说话"，不应跳变到别的情绪。

**idle → speaking 切换时机：** TTS 开始播放时切到 speaking，TTS 结束后切回 idle。切换点与 TTS 适配层的 SSML 构建同一时机（见 13.9），确保画面和声音同步。

### 14.5 组内强度调制

同一动作组覆盖了多个情绪梯度（如 angry 组覆盖 annoyed → furious），通过以下参数在组内区分：

| 参数 | 作用 | 范围 | 示例 |
|------|------|------|------|
| `playbackRate` | 视频播放速率 | 0.5–1.5 | shut_down 0.6, furious 1.3 |
| `shakeIntensity` | 身体抖动幅度（CSS/PixiJS 叠加） | 0–1 | fearful=0.3, very_fearful=0.7 |
| `swayAmplitude` | 身体前后/左右晃动幅度 | 0–1 | anxious=0.3, very_anxious=0.6 |

默认值均为 0（不叠加效果）。CSS transform 或渲染层叠加实现，不需要额外视频素材。

**注意：** `shakeIntensity` 和 `swayAmplitude` 是前端渲染层的叠加效果，不对应独立的视频文件。如果某些平台（如 app-training 轻量端）不支持渲染叠加，可忽略这两个参数，仅使用 `playbackRate`。

### 14.6 切换策略

**切换时机：** 与 TTS 同步——每轮 SP 回复开始前，状态机根据 `getOutputState()` 决定当前动作组，TTS 播放期间保持该组 speaking 变体，播放结束后切回 idle。

**切换规则：**

```
当前动作组 → 新动作组:
  相同组 → 不切换（避免同组内 annoying→furious 产生不必要的素材跳变）
  不同组 → 0.3s 淡入淡出切换
```

不采用"每句话都切"的策略——太频繁的素材切换会产生视觉跳变，破坏沉浸感。同一动作组内仅靠强度参数区分，不切换素材文件。

### 14.7 函数接口

```javascript
function getVideoCommand() {
  const state = getOutputState()
  const v = vector

  // ── 动作组映射 ──
  const group = VIDEO_ACTION_MAP[state] || 'calm'

  // ── 强度参数 ──
  let playbackRate = 1.0
  let shakeIntensity = 0
  let swayAmplitude = 0

  // 组内梯度调节
  if (group === 'angry') {
    if (state === 'furious') playbackRate = 1.3
    else if (state === 'annoyed') playbackRate = 1.1
  }
  if (group === 'fearful') {
    if (state === 'shut_down') playbackRate = 0.6
    else if (state === 'very_fearful') shakeIntensity = 0.7
    else if (state === 'fearful') shakeIntensity = 0.3
  }
  if (group === 'sad') {
    if (state === 'broken') { playbackRate = 0.7; shakeIntensity = 0.5 }
    else if (state === 'sad') playbackRate = 0.85
  }
  if (group === 'tense') {
    if (state === 'very_anxious') { swayAmplitude = 0.6; playbackRate = 1.2 }
    else if (state === 'anxious') swayAmplitude = 0.3
  }

  return {
    group,
    playbackRate: clamp(playbackRate, 0.5, 1.5),
    shakeIntensity: clamp(shakeIntensity, 0, 1),
    swayAmplitude: clamp(swayAmplitude, 0, 1)
  }
}
```

**VIDEO_ACTION_MAP：**

| TTS 状态 | 动作组 |
|----------|--------|
| calm, relieved | `calm` |
| worried, anxious, very_anxious | `tense` |
| uneasy, fearful, very_fearful, shut_down | `fearful` |
| down, sad, broken | `sad` |
| slightly_irritated, annoyed, angry, furious | `angry` |
| in_pain | `pain` |

### 14.8 素材回退

动作视频是可选增强——不是所有病例/患者形象都有完整素材。回退链：

```
speaking 视频存在？ → 使用 speaking 变体
  → 不存在？尝试 idle 视频 → 使用 idle 变体（画面不动但情绪对）
    → 不存在？回退到静态图片（当前系统行为）
      → 图片也不存在？使用 CSS 占位图标 + 情绪色点
```

素材查找按 `usePatientImage.js` 的模式：根据 patient_info 的性别/年龄匹配最接近的可用素材，不存在则回退。具体素材路径和音色 ID 一样在适配层维护。

### 14.9 AI视频素材生成管线

素材通过 AI 工具链生成，流程为三阶段：**定妆照 → 情绪表情图 → 情绪视频**。每个角色产出一套完整的 12 段视频素材（6动作组 × idle/speaking）。

#### 14.9.1 生成流程总览

```
角色定妆照（1张，中性表情，正面，纯色背景）
    │
    │  图生图：定妆照 + 情绪提示词 → 情绪表情图
    ▼
情绪表情图（12张 = 6组 × idle/speaking）
    │
    │  图生视频：情绪图 + 运动提示词 → 情绪视频
    ▼
情绪视频（12段 = 6组 × idle/speaking，每段 10-15s 循环）
```

**为什么需要情绪表情图中转？** 直接从定妆照生视频，AI 模型无法凭空将中性脸驱动到极端表情（如嚎啕大哭、怒目圆睁），先锁定正确的面部表情再驱动身体动作，质量显著更高。

#### 14.9.2 阶段一：定妆照

每个角色一张，作为后续所有生成的一致性锚点。

| 要素 | 要求 |
|------|------|
| 表情 | 中性（不喜不悲），嘴角微平，眼神自然 |
| 构图 | 正面半身像（头+肩），纯色背景（建议浅灰 #e5e7eb） |
| 光线 | 均匀正面光，无明显阴影 |
| 年龄性别 | 按患者画像设定（青年女25、中年男45、老年男65 等） |
| 着装 | 病号服或便服，按病例设定，保持同一角色着装一致 |
| 工具 | 通义万相（阿里百炼）/ Midjourney / Stable Diffusion |

**提示词模板：**
```
半身像，{性别}{年龄段}亚洲人，中性表情，正面面对镜头，纯浅灰色背景，
均匀柔光，病号服/便服，写实摄影风格，高分辨率，皮肤纹理自然
```

#### 14.9.3 阶段二：情绪表情图（12张/角色）

以定妆照为参考图，图生图叠加情绪提示词。每张图的核心差异在**面部表情**和**头部姿态**：

| 动作组 | idle 表情 | idle 姿态 | speaking 表情 | speaking 姿态 |
|--------|----------|----------|-------------|-------------|
| **calm** | 放松，微抿嘴 | 正对镜头，自然 | 温和，嘴角微扬 | 略微侧头，似在说话 |
| **tense** | 眉头微锁，嘴唇紧抿 | 正对，但肩膀微耸 | 焦虑地皱眉，眼神闪烁 | 轻微前倾，手指小动作 |
| **fearful** | 眼睁大，瞳孔收缩 | 身体微后缩 | 嘴唇微颤，眼神躲闪 | 防御性后仰，双手护胸 |
| **sad** | 眼角下垂，嘴角向下 | 低头，肩膀塌陷 | 含泪，鼻翼微张 | 低头说话，偶有抽泣 |
| **angry** | 怒视，嘴唇紧闭 | 身体前倾，肩紧绷 | 咬牙，眉头深锁 | 手势短促有力，瞪视 |
| **pain** | 痛苦皱眉，咬牙 | 身体蜷缩/手按患处 | 疼痛中勉强应答 | 吃力说话，间断停顿 |

**图生图提示词模板（以 angry-idle 为例）：**
```
[定妆照作为参考图输入]
怒视的表情，双眉紧锁，嘴唇紧闭，下巴收紧，肩膀紧绷，
身体微微前倾，纯灰色背景，保持人物身份和着装不变，写实摄影风格
```

**关键约束：**
- 人物面部特征必须与定妆照一致（靠图生图保持 identity）
- 背景保持纯色灰色，与定妆照一致
- 同一角色同一动作组的 idle 和 speaking 着装、发型完全一致
- 生成后人工筛选：剔除表情不自然或 identity 漂移的图

#### 14.9.4 阶段三：情绪视频（12段/角色）

以对应情绪表情图为起始帧，图生视频叠加运动提示词：

| 变体 | 运动描述 | 时长 | 循环 |
|------|---------|------|------|
| **idle** | 微弱的自然呼吸起伏，偶有眨眼，身体轻微自然晃动 | 10-15s | 首尾帧接近，可无缝循环 |
| **speaking** | 自然的说话口型变化，头部随语调微动，配合情绪的身体语言 | 10-15s | 同上 |

**图生视频提示词模板（以 angry-speaking 为例）：**
```
[angry-speaking情绪图作为输入图]
正在愤怒地说话，口型快速有力地张合，伴随短促有力的手势，
头部随语气节奏微动，怒视的眼神保持，纯灰色背景，人物保持稳定不位移
```

**各组的 speaking 运动强化差异：**

| 动作组 | speaking 附加运动提示 |
|--------|---------------------|
| calm | 舒缓平稳的说话节奏，轻微点头 |
| tense | 快速语速，频繁扫视，手指不安地搓动 |
| fearful | 声音颤抖的说话感，身体微微发抖，频繁吞咽 |
| sad | 缓慢低沉，偶有抽泣耸肩，用纸巾擦眼角 |
| angry | 有力的短句节奏，瞪视，拳头时而握紧 |
| pain | 吃力断断续续的说话，不时倒吸冷气，手按痛处 |

**质量验收标准：**
- 面部表情与情绪图一致，未退化回中性
- 口型变化自然（不需对口型，只求"在说话"的观感）
- 无明显的 AI 畸形帧（手指扭曲、五官漂移）
- 循环衔接处无明显跳变
