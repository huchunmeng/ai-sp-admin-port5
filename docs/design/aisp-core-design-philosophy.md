# AI-SP 核心设计思想 V2

## 一、三速混动架构

```
          ┌─────────────────────────────────────────┐
          │              反思脑（车长）                │
          │   长期策略 · 统一决策 · 唯一档位出口         │
          │   输入: CM全貌 + 阻尼器怒值                  │
          │   输出: 档位命令（给知觉脑）                  │
          │   速度: 异步(1-3s)，跨轮生效                │
          └──────────┬──────────────────────────────┘
                     │ 单一档位命令
                     ▼
  ┌──────────┐  ┌──────────────────────────────────┐
  │  阻尼器    │  │          知觉脑（驾驶员）            │
  │  纯机械    │  │   权限最低 · 档位内执行              │
  │  快升慢降  │  │   输入: 档位 + 对话上下文            │
  │  只出数值  │  │   输出: text / va / vs / anger_delta │
  └────┬─────┘  └──────────────────────────────────┘
       │ angerLevel                     ▲
       │ (数值,                      档位命令
       │  不是档位)                     │
       └────────────→ 反思脑 ──────────┘
```

**权限层级：**
1. **反思脑（车长）** — 唯一指挥官，综合 CM + 阻尼器怒值，发档位命令
2. **阻尼器（ABS）** — 纯机械反馈，不参与决策，只输出怒值供反思脑参考
3. **知觉脑（驾驶员）** — 最低权限，在档位约束内执行

---

## 二、各模块定义

### 2.1 阻尼器（Anger Damper）

**定位：** 纯机械 ABS，不是决策者。

**规则：**
- **开门无阻：** anger_delta > 0 → 直接加，不阻拦
- **关门有阻：** anger_delta ≤ 0 → 机械慢降，`-0.3`/轮
- **输出：** 只输出 `angerLevel` 数值（0-10），不输出档位
- **不受性格影响** — ABS 是物理规则，不是心理特征
- **不识别道歉/共情** — 不依赖正则判断，只执行机械衰减

**为什么只出数值不出档位：** 阻尼器只感知愤怒，不知道恐惧、悲伤、信任、卡住。让它出档位 → 愤怒无条件覆盖一切，恐惧病人被骂变"暴怒档"——错了，应该是"防御档"（恐惧驱动的抗拒）。

### 2.2 反思脑（Reflection Brain）

**定位：** 唯一指挥官，综合全局发档位命令。

**输入：**
- CM 全貌（concern / trust / stuck / dominant / attitude / unresolved_goals）
- 阻尼器 angerLevel（数值）
- 对话事件（LLM 提取）

**职责：**
1. LLM 事件提取（分类本轮对话）→ 输入规则引擎
2. 规则引擎更新 CM（确定性，零 LLM）
3. 综合 CM + 阻尼器怒值 → 输出单一档位

**档位决策逻辑（`decideGear(cm, angerLevel, ds)` — 系统唯一档位出口）：**

**11 档体系：**

| 档位 | 触发条件 | 策略方向 |
|------|---------|---------|
| **暴怒** | angerLevel ≥ 7 | 短句回怼、威胁离开、禁止示弱 |
| **愤怒** | angerLevel ≥ 4 / stuck ≥ 6 | 冷淡带刺、质问态度、不原谅 |
| **不满** | angerLevel ≥ 1 / stuck ≥ 4 或 avoid ≥ 2 | 催促反问、冷淡、表达被敷衍的烦躁 |
| **退缩** | bad_news + trust ≤ 3 + concern ≥ 7 | 沉默、哭泣、不想说了 |
| **焦虑** | concern ≥ 9 + dominant=fear + unresolved_goals > 0 | 反复追问、急切想知道答案 |
| **不安** | concern ≥ 6 + dominant=fear + unresolved_goals > 0 | 轻度担忧、说话犹豫、试探性提问 |
| **消沉** | dominant=sadness + concern ≥ 5 | 情绪低落、话少、但不崩溃 |
| **防御** | trust ≤ 2 | 拒绝配合、回避正面回答 |
| **失望** | trust 3-5 + trust_peak ≥ 6（信任曾高但下滑） | 语气降温、不再主动配合、被辜负感 |
| **配合** | trust ≥ 7 + concern ≤ 4 | 主动提问、详细描述、正面反馈 |
| **中立** | 默认 | 问什么答什么、不主动不拒绝 |

**决策优先级（`decideGear` 实现）：**
1. angerLevel ≥ 7 → **暴怒**（阻尼器物理覆盖）
2. angerLevel ≥ 4 → **愤怒**
3. angerLevel ≥ 1 → **不满**
4. stuck ≥ 6 → **愤怒**（反思脑跨轮升级）
5. stuck ≥ 4 或 avoid ≥ 2 → **不满**
6. bad_news + trust ≤ 3 + concern ≥ 7 → **退缩**
7. concern ≥ 9 + dominant=fear + unresolved_goals > 0 → **焦虑**
8. concern ≥ 6 + dominant=fear + unresolved_goals > 0 → **不安**
9. trust ≤ 2 → **防御**
10. dominant=sadness + concern ≥ 5 → **消沉**
11. trust ≥ 3 && trust ≤ 5 && trust_peak ≥ 6 → **失望**
12. trust ≥ 7 + concern ≤ 4 → **配合**
13. 默认 → **中立**

**退出滞后（hysteresis）：** 进入门槛高，退出门槛低，避免情绪跳变。

| 档位 | 进入条件 | 退出条件 |
|------|---------|---------|
| 焦虑 | concern ≥ 9 | concern < 6，或(unresolved_goals空 且 concern < 8) |
| 不安 | concern ≥ 6 | concern < 4 或 unresolved_goals空 |
| 退缩 | bad_news + trust ≤ 3 + concern ≥ 7 | concern < 4 或 trust > 5 |
| 消沉 | dominant=sadness + concern ≥ 5 | dominant≠sadness 或 concern < 3 |
| 配合 | trust ≥ 7 + concern ≤ 4 | trust < 5 或 concern > 7 |
| 失望 | trust 3-5 + trust_peak ≥ 6 | trust ≥ 6 或 trust ≤ 2（分别回到配合/防御） |

**注意1：** 不支持复合档位。知觉脑输出通道只有文字+TTS音色+固定视频动作，复合策略的细微差异无法体现。单一档位选中后，LLM 在策略文本约束内发挥即可。

**注意2：** `trust_peak` 是 CM 新增字段：记录对话历史中 trust 达到的最高值，用于判断信任是否从高处下滑（失望 vs 本来就不信任）。

**注意3：** 不安、消沉、失望三者填补了 8 档体系的情绪空白——轻度恐惧（担忧但没到焦虑）、轻度悲伤（低落但没到退缩）、信任降温（被辜负但没到防御）。

**时序保证：** 反思脑异步运行（1-3s 完成），远短于学生打字间隔（5-30s），下一轮前必定完成。

### 2.2.1 档位 → 视听通道直连（`getGearAV`）

每个档位直接映射到 va（视频动作）/ vs（语音风格）候选菜单，不再经过 intent+derived_state 中间层。

**va 标签体系（9 种，按肉眼可辨识的说话动态区分）：**

va 标签选择原则：文生视频中，静态待机帧的差异往往不够大，关键区分点在**说话时的动态表达**。两个标签如果说话动态无法肉眼区分就合并。

```
愤怒系: angry_intense(怒吼+大幅手势) / angry(强硬质问)
恐惧系: fearful(瞪眼后退、惊慌)
悲伤系: sad(低头低沉) / broken(抽泣断续)
焦虑系: anxious(搓手抿嘴、急切追问)  [焦虑≠恐惧，身体方向相反]
防御系: defensive(双臂交叉、身体后仰、眼神回避)
配合系: warm(微笑、身体前倾、眼神交流)
中立系: neutral(面无表情、正常站立)
```

未变，9 种 va 标签覆盖 11 档位——新档位复用现有标签：

**11 档位 × 候选映射：**

| 档位 | va 候选 | 待机 | 说话动态 | vs 候选 |
|------|--------|------|---------|--------|
| **暴怒** | `angry_intense` | 面部扭曲、身体前倾 | 怒吼、大挥手、指向前方 | `very_loud_fast` |
| **愤怒** | `angry` | 皱眉瞪视、身体微紧 | 强硬质问、昂头 | `loud_fast`, `cold` |
| **不满** | `angry` | 嘴角下拉 | 冷淡带刺、叹气翻白眼、不与人眼神接触 | `cold`, `defensive` |
| **焦虑** | `anxious`, `fearful` | 搓手/眼神游移 | 急切追问、身体前倾（非攻击）、声音发抖 | `shaky`, `very_shaky` |
| **不安** | `anxious` | 双手交握、眼神游移 | 犹豫提问、身体微前倾 | `slightly_tense`, `shaky` |
| **退缩** | `sad`, `broken` | 低头垂肩/抱头蜷缩 | 低沉缓慢/抽泣断续 | `soft_slow`, `broken` |
| **消沉** | `sad` | 低头、肩膀松垮 | 说话慢、叹气、无眼神交流 | `soft_slow` |
| **防御** | `defensive` | 双臂交叉、身体后仰 | 冷淡回应、眼神回避、拒绝姿态 | `defensive`, `cold` |
| **失望** | `defensive` | 表情冷淡、身体微后仰 | 语气降温、简短回应、不再主动 | `cold`, `flat` |
| **配合** | `warm` | 放松、微笑 | 点头、身体前倾、眼神交流 | `warm`, `normal` |
| **中立** | `neutral` | 正常站立、面无表情 | 平淡回答、无多余动作 | `flat`, `normal` |

知觉脑 LLM 从候选菜单中选择一个返回，不再自行决定情绪方向。

**知觉脑选择空间：**

11 个档位中，va 候选大多为 1 个（部分有 2 个），vs 候选 1-2 个。实际选择空间很小——档位基本锁死了视听表现。这是刻意设计的：反思脑已经做了全局决策，知觉脑不需要在视听层面"再选一遍"。焦虑/不安/退缩保留 2 个 va 候选，给知觉脑根据对话强度选择的空间。

**突发豁免机制：**

档位基于 N-1 轮的 CM + damper，反映的是跨轮状态。但学生**本轮**可能发生与档位预期相反的事件。两种场景：

- **突发攻击**：学生在中立/配合档时突然辱骂、嘲讽 SP
- **坏消息告知**：学生在本轮告知重大负面诊断结果（晚期癌症、严重预后等）

这两类事件不由正则检测——**知觉脑自己读学生消息判断**。如果触发，知觉脑有权在策略文本约束范围内做出即时反应：

```
配合 → 被攻击/坏消息 → 可至 失望/防御（停止配合、降温），禁到攻击
中立 → 被攻击 → 可至 防御（冷淡反问），禁到攻击
中立 → 坏消息 → 可至 消沉/不安（低落/担忧），禁到退缩/崩溃
焦虑 → 坏消息加重 → 可至 退缩
不安 → 坏消息 → 可至 焦虑
失望 → 被攻击 → 可至 防御
防御 → 被攻击 → 可至 更强防御（更冷更短），禁到攻击
```

**注意：** 不满/愤怒/暴怒档本身已对冲突有预期，不需豁免。消沉/退缩档本身的反应方向与坏消息一致，不需额外豁免。

这个机制不修改档位，不影响跨轮状态。下一轮 damper 被 anger_delta 推高后，反思脑会自然输出更高级档位。恢复配合需要多轮信任重建（阻尼器慢降 + trust 逐步提升），符合"伤害容易修复难"的人性规律。

### 2.2.2 voice_style → CosyVoice 韵律参数映射

 CosyVoice `_v3` 音色不支持文本指令（instruction），仅靠 `rate`（语速）/ `pitch`（音调）/ `volume`（音量）三个韵律参数表达情绪。映射表定义于 `apps/training/public/data/patient-voice-config.json` → `cosyvoice-v3-flash.emotionParams`：

| voice_style | rate | pitch | volume | 对应档位 | 听感描述 |
|------------|------|--------|--------|---------|---------|
| `warm` | 1.0 | 1.05 | 50 | 配合 | 温和带关切，略上扬 |
| `flat` | 0.95 | 0.95 | 45 | 中立 | 平淡，不带情绪波动 |
| `normal` | 1.0 | 1.0 | 50 | 配合/中立回退 | 正常关切 |
| `slightly_tense` | 1.1 | 1.05 | 55 | 不安 | 略急促，试探性 |
| `loud_fast` | 1.4 | 1.15 | 75 | 愤怒 | 语速快、声音大 |
| `very_loud_fast` | 1.6 | 1.25 | 90 | 暴怒 | 极快极响，接近吼叫 |
| `cold` | 0.9 | 0.9 | 40 | 不满/防御/失望 | 慢、低沉、冷淡 |
| `shaky` | 1.25 | 1.15 | 55 | 焦虑/不安 | 快而不稳，颤抖感 |
| `very_shaky` | 1.5 | 1.25 | 60 | 焦虑(极端) | 极快极抖，惊恐 |
| `soft_slow` | 0.7 | 0.85 | 35 | 退缩/消沉 | 慢、轻、消沉 |
| `defensive` | 0.95 | 0.95 | 42 | 防御/不满 | 略慢、收着、抗拒 |
| `vulnerable` | 0.8 | 0.9 | 30 | 退缩(极端) | 虚弱颤抖带哭腔 |
| `broken` | 0.6 | 0.8 | 22 | 退缩(崩溃) | 极慢极轻、断断续续 |

**注意：** Qwen-TTS 映射（`VOICE_INSTRUCTIONS` 中文指令）保留不受影响，此处仅覆盖 CosyVoice 链路。

### 2.2.3 video_action → 文生视频提示词映射

视觉系统采用文生图→图生视频管线，每个 `video_action` 对应待机/说话两个视频。提示词聚焦于**肉眼可辨识的动态差异**：

| video_action | 对应档位 | 待机(idle) 提示词 | 说话(speaking) 提示词 |
|-------------|---------|------------------|---------------------|
| `angry_intense` | 暴怒 | 面部扭曲、身体大幅前倾、手指指向屏幕外 | 怒吼、大挥手、头部快速转动、身体前后晃动 |
| `angry` | 愤怒/不满 | 皱眉、嘴角下拉、身体微紧 | 强硬质问、昂头、偶尔叹气翻白眼 |
| `anxious` | 焦虑/不安 | 搓手、抿嘴、眼神快速游移、坐立不安 | 急切追问、身体前倾、手部不安地摆动 |
| `fearful` | 焦虑(回退) | 眼神惊恐、身体微缩后仰 | 惊慌后退、摇头、手护胸前 |
| `sad` | 退缩/消沉 | 低头垂肩、眼神空洞、身体松垮 | 低沉缓慢开口、身体轻微晃动、偶尔叹气 |
| `broken` | 退缩(极端) | 抱头蜷缩、全身无力、面部埋入手中 | 抽泣断续、肩膀抖动、无法抬头 |
| `defensive` | 防御/失望 | 双臂交叉、身体后仰、眼神疏离、嘴唇紧抿 | 冷淡回应、摇头、眼神回避、拒绝姿态 |
| `warm` | 配合 | 放松站立、微笑、双手自然垂放 | 点头、身体微前倾、频繁眼神交流、温和手势 |
| `neutral` | 中立 | 正常站立、面无表情 | 平淡回答、无多余动作 |

**设计原则：**
- 两个标签如果在待机或说话中任一环节肉眼分不出 → 合并
- `angry_intense` vs `angry`：分。暴怒的大挥手指向前方 vs 愤怒的昂头质问，动态差异明显
- `angry` vs `angry_mild`（已删除）：不分。不满的冷淡叹气 vs 愤怒的强硬质问，区别在语音而非视频
- `anxious` vs `fearful`：分。焦虑是搓手抿嘴的前向紧张，恐惧是瞪眼后退的回避反应，身体方向完全相反
- `fearful` vs `fearful_intense`（已删除）：不分。恐惧的强度差异视频里分不出来
- `warm` vs `neutral`：分。微笑+前倾 vs 面无表情+直立，待机就能明显区分
- `defensive` vs `neutral`：分。交叉手臂+后仰 vs 正常站立，待机就能明显区分

### 2.3 知觉脑（Perception Brain）

**定位：** 驾驶员，权限最低。

**输入：** 反思脑的统一档位命令 + 性格提示词 + 对话上下文

**输出：** text / video_action / voice_style / anger_delta

**约束：** 在档位约束内自由发挥，不能违背档位精神。档位是策略文本（如"防御档：回避正面回答，表达不安"），不是数值。

## 三、场景初始化

会话创建时，分类 LLM（`qwen-plus`, `temp=0.3`）读取病例角色描述/症状文本，输出：

```json
{
  "sceneType": "坏消息告知",
  "initialTrust": 3,
  "initialConcernIntensity": 8,
  "initialAnger": 0
}
```

不同场景有不同的情绪起点。30s 超时，失败有默认回退值。

## 四、性格系统

性格仅通过**提示词**影响知觉脑和反思脑的 LLM 调用：

- 知觉脑：性格注入系统提示词 → 影响 text / va / vs
- 反思脑：性格注入事件提取提示词 → 影响事件分类
- 阻尼器：不受性格影响（ABS 是物理规则）

四种性格：`hot-tempered` / `stoic` / `anxious` / `suspicious`

---

## 五、关键词检测位置

仅在 **LLM 调用之后** 运行：

- `detectInsultTrigger`：辱骂关键词 → 阻尼器至少 +3（安全兜底）
- `classifyIntentByRule`：不参与阻尼器衰减

**原则：** 关键词检测不挡在知觉脑（LLM）前面，不造成延迟。

## 六、数据流（每轮对话）

```
1. 收到学生消息
2. A/B/closure trigger 检测（角色一致性硬规则，LLM前）
3. computeDerivedState(session.cm) → ds（CM → 主导情绪/态度）
4. session.damper.getLevel() → damperLevel（阻尼器纯数值）
5. decideGear(session.cm, damperLevel, ds) → effectiveGear（反思脑唯一档位出口）
6. getGearAV(effectiveGear) → avCandidates（档位 → 视听菜单）
7. getPerceptionPersona(session.personality) → personalityPrompt（性格提示词）
8. buildSystemPrompt({ effectiveGear, avCandidates, personalityPrompt, ... }) → 系统提示词
9. 知觉脑 LLM 调用 → text / va / vs / anger_delta
10. 阻尼器同步更新（anger_delta > 0 → 升；≤ 0 → -0.3）
11. 反思脑异步启动（事件提取 → CM更新 → 档位预计算）
12. 返回响应
```

步骤 11 在步骤 12 之后完成，不影响当前响应。下一轮步骤 3-5 读取步骤 11 的更新结果。

关键：档位决策只在步骤 5 一处发生（`decideGear`），阻尼器只提供 angerLevel 数值，不参与档位判断。

**时序：**
```
轮次 N-1 结束:
  ├─ 阻尼器同步更新 angerLevel (即时，0ms)
  └─ 反思脑异步启动 (fire-and-forget)
       ├─ LLM事件提取 (1-3s)
       └─ CM规则更新 (0ms)
       
学生打字中... (5-30s)  ← 反思脑早已完成

轮次 N 开始:
  ├─ CM 已更新 ✅
  ├─ damper.angerLevel 已更新 ✅
```
