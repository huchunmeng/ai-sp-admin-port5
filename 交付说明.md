# AI-SP 标准化病人平台 — 本地启动指南

## 环境要求

- Node.js 18+
- npm 9+

## 快速启动

```bash
# 1. 解压后进入项目目录
cd ai-sp-admin

# 2. 安装依赖
npm install

# 3. 启动开发服务器（五端并行）
npm run dev
```

启动后各端访问地址：

| 端 | 地址 |
|----|------|
| 训练端 | http://localhost:5001 |
| 管理端 | http://localhost:5002 |
| 考试端 | http://localhost:5003 |
| App训练端 | http://localhost:5004 |
| 运营平台 | http://localhost:5005 |

## LLM 功能配置（可选）

默认使用降级/Mock模式，如需真实LLM对话和AI生成功能：

1. 在 `apps/training/` 下创建 `.env.local`：
```
LLM_API_KEY=你的阿里云百炼API密钥
LLM_MODEL=qwen-turbo
```

2. 在 `apps/admin/` 下创建 `.env.local`：
```
VITE_ENABLE_AI_GENERATE=true
AI_GENERATE_API_KEY=你的AI API密钥
```

不配置也不影响基本功能测试。

## 单独启动某一端

```bash
npm run dev:training   # 仅训练端 5001
npm run dev:admin      # 仅管理端 5002
npm run dev:exam       # 仅考试端 5003
npm run dev:app        # 仅App训练端 5004
npm run dev:ops        # 仅运营平台 5005
```

## 常见问题

**Q: 启动后页面空白？**
确认是否进入了正确的端口（如训练端是 5001 而非 5002）。

**Q: 病例列表为空？**
确认 `apps/admin/public/data/cases/` 目录下存在 `*-basic.json` 文件。

**Q: 端口被占用？**
修改对应 `apps/<name>/vite.config.js` 中 `server.port` 的值。
